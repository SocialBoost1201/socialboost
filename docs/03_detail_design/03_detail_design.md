## 共通レイアウト詳細設計

### ページ全体構造

全ページ共通で以下の基本構造を持つ。

1. 共通ヘッダー
2. ページ固有メインコンテンツ
3. ページ下部CTAまたは補助導線
4. 共通フッター

### 共通横幅設計方針

- メインコンテンツは可読性を確保できる最大幅を持つ
- 実績グリッドやビジュアルセクションは広めの表示領域を許容する
- 本文主体セクションは読みやすさ重視の横幅に抑える
- スマホでは左右余白を維持しながら縦積みを基本とする

### 共通余白設計方針

- セクション間は明確に区切る
- 要素間の詰め込みを避ける
- 見出しと本文、本文と補足、カード同士の余白ルールを一貫させる
- 余白によって上品さと整理感を作る

## 共通ヘッダー詳細設計

### 役割

- 全ページで主要導線を明確にする
- サイト全体の信頼感を補強する
- スクロール中でも問い合わせ導線を保つ

### 表示要素

- ロゴ
- ナビゲーション
- お問い合わせ導線
- モバイルメニュー導線

### ナビゲーション項目

| 順序 | ラベル |
|:---|:---|
| 1 | サービス |
| 2 | 料金 |
| 3 | 実績 |
| 4 | 選ばれる理由 |
| 5 | お問い合わせ |

### 表示仕様

- PCでは横並び表示を基本とする
- スマホではハンバーガーメニューまたは省略メニュー形式を前提とする
- ロゴクリックでトップページへ遷移する
- お問い合わせは通常ナビ項目より強い視認性を持たせる前提とする

### 状態変化

| 状態 | 仕様 |
|:---|:---|
| 初期表示時 | 背景は透明または極めて薄い背景を基本候補とする |
| スクロール後 | 背景色付与または半透明化、影または境界線付与、高さをやや縮める演出を許容する |

### 避けること

- 重すぎる固定ヘッダー
- 過度なアニメーション
- 情報量の多すぎるヘッダー
- 会社概要など優先度の低い導線追加

### ファーストビュー詳細設計

#### 目的

- 何をしている事業かを一瞬で伝える
- 制作だけでなく、システム、アプリ、AIまで対応できることを伝える
- 第一印象で信頼感と先進性を与える

#### 構成要素

- メインコピー
- サブコピー
- 主要CTA
- 補助CTA
- ビジュアル領域

#### メインコピー要件

- 短く強いこと
- 抽象だけで終わらないこと
- 事業成果に直結する印象を持たせること

#### サブコピー要件

- Web制作、Webシステム開発、アプリ開発、AI導入支援、改善運用のような対応範囲を理解できること

#### CTA要件

| 種別 | 候補 |
|:---|:---|
| 主要CTA | お問い合わせ / 無料相談 / 相談してみる |
| 補助CTA | 実績を見る / 料金を見る / サービスを見る |

#### ビジュアル要件

- 写真主導ではなく、UI、サイト画面、ダッシュボード、アプリ画面、提案資料の要素などを主役候補とする
- 青系ブランドに合う静かな先進性を持たせる
- ロゴと喧嘩しない構成にする

#### 配置方針

| デバイス | 方針 |
|:---|:---|
| PC | コピーとビジュアルの2カラムを基本候補とする |
| スマホ | コピー、CTA、ビジュアルの縦積みを基本とする |

## アニメーション詳細設計

### 適用対象

- ファーストビュー
- セクション出現
- カードホバー
- ヘッダー状態変化
- ギャラリー表示切り替え
- 詳細画面遷移時の穏やかな変化

### 適用方針

- 導線を補助する
- 静かな高級感を支える
- 読みやすさを邪魔しない
- 表示速度を損なわない

### 禁止事項

- 常時動き続ける装飾
- 派手なバウンド
- 過度なズーム
- 大きすぎる視差演出
- 読解の邪魔になる切り替え
