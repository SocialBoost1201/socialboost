# SocialBoost 公式Webサイト 機能設計書

---

## 文書目的

本書は、SocialBoost 公式Webサイトに必要な各機能について、機能の目的、利用者、対象ページ、入力、出力、画面上の挙動、関連データ、例外時の扱い、将来拡張方針を整理し、後続のUI設計、技術設計、テスト設計、運用設計の基準とすることを目的とする。

---

## 対象プロジェクト

| 項目 | 内容 |
|:---|:---|
| 名称 | SocialBoost 公式Webサイト |
| プロジェクトフォルダ | `socialboost` |

---

## 前提

本フェーズでは実装を行わない。
本書は設計文書であり、コード生成、UI実装、依存追加、ルーティング実装、管理画面実装、デプロイは行わない。
ユーザー本人が明示的に実装許可を出すまで、実装フェーズへ進まない。

---

## 機能設計方針

本サイトの機能は、派手さや過剰なギミックではなく、営業力、信頼形成、閲覧性、相談しやすさ、将来拡張性を重視する。
特に以下を重要視する。

1. 何をしている事業かを短時間で理解できること
2. サービス内容と対応範囲を迷わず確認できること
3. 実績を一覧から詳細へ自然に閲覧できること
4. 実績詳細で画像とPDFを整理して閲覧できること
5. 料金の目安を把握できること
6. 問い合わせに自然につながること
7. 実績やサービスが増えても構造破綻しないこと

## F-01：共通ヘッダー表示機能

| 項目 | 内容 |
|:---|:---|
| 機能ID | F-01 |
| 目的 | 全ページで主要導線を一定品質で提供する |
| 利用者 | 全訪問者 |
| 対象ページ | 全ページ |
| 入力 | なし |
| 出力 | ヘッダーUI一式 |

### 表示要素

- ロゴ
- サービス
- 料金
- 実績
- 選ばれる理由
- お問い合わせ

### 基本挙動

- ロゴクリックでトップページへ遷移する
- 各ナビゲーション項目クリックで対応ページまたは対応セクションへ遷移する
- お問い合わせ導線は通常項目より視認性を高める前提とする

### 状態

- 通常表示
- スクロール後表示
- モバイル表示

### 例外

- 該当ページが未作成の場合は仮リンクまたは非活性扱いを後続設計で決定する

## 非機能上の注意

- 全機能は表示速度を損なわないことを前提とする
- 重いアニメーションや過剰なファイル読込を前提にしない
- スマホ表示を軽視しない
- 一覧と詳細の責務を混在させない
- 画像とPDFを混在表示しない

## テスト観点への引継事項

- ヘッダー導線が全ページで正しく機能するか
- サービス一覧から詳細へ遷移できるか
- 料金ページで参考レンジが正しく表示されるか
- 実績一覧から詳細へ遷移できるか
- 実績詳細で画像一覧とPDF一覧が分離表示されるか
- 関連実績が適切に表示されるか
- お問い合わせフォームで入力エラーが適切に案内されるか
- 送信完了状態が明確に表示されるか
- CTAが各主要ページで機能しているか
- PC、タブレット、スマホで表示が破綻しないか
