# SocialBoost 公式Webサイト SEO設計書

---

## 文書目的

本書は、SocialBoost 公式Webサイトにおける検索流入獲得のためのSEO方針、ページ別の検索意図設計、キーワード設計、内部リンク設計、メタ情報設計、見出し設計、コンテンツ重複防止方針、実績ページのSEO活用方針を定義し、後続のコピー設計、実装設計、運用設計、改善施策の基準とすることを目的とする。

## 前提

## SEO設計の基本思想

SocialBoost のSEOは、単なる流入数の拡大だけを目的としない。
本サイトでは、指名検索、比較検討検索、サービス検討検索、事例確認検索を適切に受け止め、**問い合わせにつながる質の高い流入を獲得する** ことを重視する。
無差別な記事量産ではなく、ページごとの役割分担、検索意図との一致、実績とサービス内容の整合、内部リンクによる理解導線を優先する。

---

## SEO設計の目的

1. SocialBoost のブランド名検索に確実に対応すること
2. Web制作、LP制作、Webシステム開発、アプリ開発、AI導入支援などの主要サービス検索に対応すること
3. 実績ページを信頼形成と検索流入の両面で活用すること
4. トップページに情報を集約しすぎず、下層ページにも独立した検索価値を持たせること
5. 問い合わせに近い検索意図の受け皿を明確に持つこと
6. 将来的な記事追加や資料追加にも耐えられるSEO基盤を作ること

---

## SEOの基本方針

1. ページごとの役割を明確に分ける
2. 一覧ページと詳細ページを役割分担する
3. サービス詳細ページを主要な検索受け皿にする
4. 実績詳細ページを事例検索と信頼形成の資産にする
5. 料金ページは価格不安の解消と比較検討需要の受け皿にする
6. お問い合わせページは行動ページとして整理し、無理に情報過多にしない
7. 同じ説明文の使い回しによる重複を避ける
8. 見出し、本文、内部リンク、メタ情報の整合性を保つ
9. 更新しやすく、改善しやすい構造を優先する

## キーワード設計方針

キーワード設計は、無理にビッグワードだけを追わず、意図の近いキーワードをページごとに整理して対応する。

### キーワードの階層

| 階層 | 内容 |
|:---|:---|
| 第1階層 | ブランドキーワード（SocialBoost、SocialBoost 公式、SocialBoost Web制作） |
| 第2階層 | サービスキーワード（Web戦略設計、LP制作、Webシステム開発、アプリ開発、AI導入支援 等） |
| 第3階層 | 比較検討キーワード（ホームページ制作 料金、LP制作 費用、システム開発 相談 等） |
| 第4階層 | 事例キーワード（Web制作 実績、LP制作 事例、アプリ開発 実績 等） |
| 第5階層 | 地域性キーワード（横浜 Web制作、神奈川 ホームページ制作 等）※事業実態と整合する場合に限定 |

### キーワード配置方針

| ページ | 配置方針 |
|:---|:---|
| トップページ | ブランド名、総合支援内容、主要サービスワードの自然な配置 |
| サービス詳細 | 各サービス固有ワード、対象顧客ワード、課題ワード |
| 料金ページ | 費用、料金、相場、相談などの比較検討ワード |
| 実績一覧 | 実績、事例、制作実績、開発実績などの一覧意図ワード |
| 実績詳細 | 案件内容に即したキーワード、業種名、カテゴリ名、支援内容 |

## 見出し設計方針

### 基本原則

- 1ページに主見出しは1つ
- 見出しは本文内容と一致させる
- 見出しだけ読んでもページ構造が理解できるようにする
- キーワードを不自然に詰め込まない

### 見出しの役割

| 階層 | 役割 |
|:---|:---|
| 主見出し | ページ全体のテーマ提示 |
| 第2階層見出し | セクションの意味区切り |
| 第3階層見出し | 補足区切り、一覧項目のグルーピング |

### 避けること

- 同じ表現の繰り返し
- 抽象的すぎる表現だけ
- キーワード羅列
- 内容と一致しないキャッチー表現だけの見出し

## 本文設計方針

### 基本原則

- 本文はSEOのためだけに書かない
- 読み手にとって役立つ順で整理する
- 抽象論だけで終わらせない
- 一覧ページでは短く、詳細ページでは具体的に書く

### ページ別本文方針

| ページ | 方針 |
|:---|:---|
| トップページ | 要約、入口、導線中心 |
| サービス詳細 | 対象顧客、課題、提供内容、対応範囲、進行フロー、関連実績を具体的に |
| 料金ページ | 価格の考え方、参考レンジ、変動要因、不安解消を整理 |
| 実績詳細 | 案件概要、課題、対応範囲、実施内容、成果または狙いを具体的に |

### 避けること

- トップページに全情報を詰め込むこと
- 各ページで同じ文章を再利用し続けること
- 不自然なキーワード反復
- 本文が画像やPDF任せになること

## 内部リンク設計方針

内部リンクはSEOと回遊性の両面から重要とする。ただし過剰に貼らず、文脈に合うリンクを設置する。

### 主要内部リンク構造

```
トップページ       → サービス一覧 / 料金 / 実績一覧 / 選ばれる理由 / お問い合わせ
サービス詳細       → 関連実績 / 料金 / お問い合わせ / 他サービス
料金ページ         → サービス詳細 / お問い合わせ / 実績
実績一覧           → 実績詳細
実績詳細           → 関連実績 / 関連サービス / お問い合わせ
選ばれる理由       → 実績一覧 / サービス一覧 / お問い合わせ
お問い合わせ       → プライバシーポリシー / サービス / 料金 / 実績
```

### 原則

- 文脈のない大量リンクを置かない
- 同一ページ内で同じリンク先を増やしすぎない
- 行動導線とSEO導線を分けて考える
- 一覧は一覧、詳細は詳細の役割を壊さない

## メタ情報設計方針

### タイトル設計方針

- ページごとに一意にする
- ブランド名とページ役割を両立する
- 短く分かりやすくする
- サービスページではサービス名を主軸にする
- 実績ページでは案件名と実績の文脈を持たせる

### ディスクリプション設計方針

- ページ内容を要約する
- 問い合わせ文言だけに寄せない
- 読み手がクリック後に得られる情報を簡潔に示す
- 各ページで使い回さない

### 重点ページ

トップページ / サービス詳細ページ / 料金ページ / 実績一覧ページ / 実績詳細ページ / 選ばれる理由ページ / お問い合わせページ

## コンテンツ重複防止方針

### ページ別の役割特化

| ページ | 特化内容 |
|:---|:---|
| トップページ | 要約 |
| サービス一覧 | 比較 |
| サービス詳細 | 具体説明 |
| 実績一覧 | 一覧比較 |
| 実績詳細 | 案件固有情報 |
| 料金ページ | 価格情報 |

### 重複しやすい領域

- トップページとサービス一覧
- サービス一覧とサービス詳細
- 実績一覧と実績詳細
- 料金ページとサービス詳細
- 選ばれる理由とトップページ

### 重複回避ルール

- 同じ文章をコピペしない
- 一覧ページでは短文化する
- 詳細ページでは固有内容を追加する
- 理由ページは思想と差別化に特化する
- 価格ページは価格思想と参考レンジに特化する

## SEO運用方針

| 更新頻度 | 対象 |
|:---|:---|
| 高頻度 | 実績詳細ページ、実績一覧の要約、サービスページ補足、料金補足、メタ情報改善 |
| 中頻度 | 内部リンク調整、タイトル調整、導線改善、カテゴリ整理 |
| 低頻度 | トップページ骨格、サイト構造、主要URL |

### 運用ルール

- キーワード先行でページを量産しない
- ページ追加時は役割と検索意図を定義してから作る
- 既存ページで足りるなら新規ページを増やさない
- 重複表現を定期的に見直す
- 検索流入だけでなく問い合わせ導線まで見る

---

## SEO上で避けること

- トップページへの情報集中
- 同じ文の大量再利用
- 実績ページの画像依存
- 価格ワードの安売り訴求
- 薄いサービス詳細ページ
- 形だけのカテゴリページ乱立
- 意味のない記事量産
- 英語混じりの分かりにくいURL
- 意図不明なページ追加
