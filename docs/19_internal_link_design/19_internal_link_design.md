# SocialBoost 公式Webサイト 内部リンク設計書

---

## 文書目的

本書は、SocialBoost 公式Webサイトにおける内部リンクの役割、配置方針、ページ間接続ルール、導線優先順位、SEOおよびGEO観点での設計原則を定義することを目的とする。
本書は、サイトマップ設計書、情報設計書、各ページ設計書、SEO設計書、GEO設計書を補完し、後続のUI設計、実装設計、改善運用、テスト設計の基準とする。

## 前提

本フェーズでは実装を行わない。
本書は内部リンクの設計文書であり、リンク実装、ルーティング作成、アンカー設置、パンくず実装、ナビゲーション実装、デプロイは行わない。
ユーザー本人が明示的に実装許可を出すまで、実装フェーズへ進まない。

---

## 内部リンク設計の基本思想

SocialBoost の内部リンクは、単にページ同士を機械的につなぐものではない。
役割は、理解を深めること、比較検討を助けること、相談導線をつなぐこと、検索エンジンと生成AIにページ関係を伝えることにある。

内部リンクは以下の流れを自然に支える構造で設計する：
- 何をしている事業か → 何ができるか → どんな実績があるか → どのくらいの価格感か → なぜ選ばれるか → どう相談できるか

---

## 内部リンク設計の目的

1. ユーザーが迷わず次に読むべきページへ進めること
2. トップページだけに依存しない回遊構造を持たせること
3. サービス、料金、実績、問い合わせを相互に接続し、比較検討しやすくすること
4. 一覧ページと詳細ページの関係を明確にすること
5. SEO上のページ評価伝達を助けること
6. GEO上のページ関係の理解を助けること
7. 将来的なページ追加にも耐えられるルールを持つこと

---

## 内部リンク設計の基本方針

1. ページ文脈に合うリンクだけを置く
2. リンク先の役割が明確であること
3. 一覧ページは詳細ページへの入口として機能させる
4. 詳細ページは関連詳細と問い合わせへつなぐ
5. 価格ページは価格確認だけで終わらせず、サービス理解と相談へ戻す
6. 問い合わせページは補助導線を持つが、主役はフォームであることを保つ
7. 過剰なリンク密度にしない
8. 同一ページ内で同じリンク先を不必要に繰り返さない
9. 見出しや文脈とリンク先の内容を一致させる
10. ナビゲーションリンクと本文リンクの役割を分ける

---

## 内部リンクの役割分類

| 分類 | 役割 | 例 |
|:---|:---|:---|
| 第1分類：主導線リンク | サイト全体の主要行動を支える | サービス / 料金 / 実績 / 選ばれる理由 / お問い合わせ |
| 第2分類：理解深化リンク | 詳細理解を助ける | サービス詳細 / 実績詳細 / 関連実績 / 関連サービス |
| 第3分類：比較検討リンク | 判断材料をつなぐ | サービス詳細↔料金 / 実績詳細→関連実績 / 選ばれる理由→実績 |
| 第4分類：行動誘導リンク | 問い合わせへつなぐ | お問い合わせ / 無料相談 / 相談してみる |
| 第5分類：補助リンク | 法務・将来拡張 | プライバシーポリシー / FAQ / 資料ページ |

---

## リンク配置の原則

| 配置種別 | 方針 |
|:---|:---|
| ナビゲーションリンク | ヘッダーとフッターに置く常設導線。本文理解の代替にはしない |
| 本文リンク | ページ文脈に沿って配置。リンク文言だけで遷移先が想像できること |
| カードリンク | 一覧ページや関連実績で使用。カード全体クリックを基本候補 |
| CTAリンク | 相談・実績閲覧・料金確認などの行動導線。主従関係を明確にする |

## ページ群ごとの内部リンク方針

### トップページ

| 項目 | 内容 |
|:---|:---|
| 必須リンク先 | サービス一覧 / 料金ページ / 実績一覧 / 選ばれる理由 / お問い合わせ |
| 推奨補助リンク | 代表的なサービス詳細 / 代表的な実績詳細 |
| 方針 | 入口としての交通整理。トップだけで完結させず下層へ送る |

### サービス一覧ページ

| 項目 | 内容 |
|:---|:---|
| 必須リンク先 | 各サービス詳細 / 関連実績候補 / お問い合わせ |
| 推奨補助リンク | 料金ページ / 選ばれる理由 |
| 方針 | 一覧は詳細への入口に徹する。課題別導線からもサービス詳細へ接続 |

### サービス詳細ページ

| 項目 | 内容 |
|:---|:---|
| 必須リンク先 | 料金ページ / お問い合わせ / 関連実績 / 必要に応じて他サービス詳細 |
| 推奨補助リンク | サービス一覧 / 選ばれる理由 |
| 方針 | 料金導線と実績導線は必須。他サービスへの導線は必要な場合に限定 |

### 料金ページ

| 項目 | 内容 |
|:---|:---|
| 必須リンク先 | お問い合わせ / サービス一覧 / 各サービス詳細 |
| 推奨補助リンク | 実績一覧 / 代表的な実績詳細 |
| 方針 | 料金だけ見て離脱させない。サービス内容か相談へ戻す |

### 実績一覧ページ

| 項目 | 内容 |
|:---|:---|
| 必須リンク先 | 各実績詳細 / お問い合わせ |
| 推奨補助リンク | 料金ページ / サービス一覧 |
| 方針 | 実績カードは詳細へ直接つなぐ。一覧は比較、詳細は理解の役割分担を守る |

### 実績詳細ページ

| 項目 | 内容 |
|:---|:---|
| 必須リンク先 | 関連実績 / お問い合わせ / 必要に応じて関連サービス |
| 推奨補助リンク | 実績一覧 / 料金ページ / 選ばれる理由 |
| 方針 | 案件本文の最後で自然に相談導線へつなぐ。関連実績は回遊補助 |

### 選ばれる理由ページ

| 項目 | 内容 |
|:---|:---|
| 必須リンク先 | 実績一覧 / サービス一覧 / お問い合わせ |
| 推奨補助リンク | 料金ページ / 代表的な実績詳細 |
| 方針 | 理由を読んだ後、根拠として実績を見たくなる流れを作る |

### お問い合わせページ

| 項目 | 内容 |
|:---|:---|
| 必須リンク先 | プライバシーポリシー |
| 推奨補助リンク | サービス一覧 / 料金ページ / 実績一覧 / 選ばれる理由 |
| 方針 | 主役はフォーム。補助リンクは逃げ道として配置。フォームより目立たせない |

### プライバシーポリシーページ

| 項目 | 内容 |
|:---|:---|
| 推奨リンク先 | トップページ / お問い合わせページ |
| 方針 | 最小限でよい。主回遊の中心にはしない |

## 主導線リンク設計

### ヘッダー導線

サービス / 料金 / 実績 / 選ばれる理由 / お問い合わせ

### フッター導線

トップ / サービス / 料金 / 実績 / 選ばれる理由 / お問い合わせ / プライバシーポリシー

### 設計原則

- ヘッダー項目は増やしすぎない
- 現時点で会社概要を主ナビに置かない
- お問い合わせを主導線として明確にする
- フッターはヘッダーより少し広くてもよいが情報過多にはしない

---

## 本文リンク設計

### ルール

- リンク文言だけで遷移先が分かること
- 同じ文脈で複数リンクを乱立させないこと
- 段落全体をリンクにしないこと
- 1段落内のリンク数を増やしすぎないこと

### 望ましい文言例

- 詳しい支援内容はサービスページでご確認ください
- 関連する実績はこちら
- 参考レンジは料金ページでご案内しています
- このような案件のご相談はこちら

### 避ける文言

「こちら」「詳細」「もっと見る」だけで意味が伝わらないものの乱用

## カードリンク設計

| カード種別 | リンク先 |
|:---|:---|
| サービスカード | サービス詳細ページ |
| 実績カード | 実績詳細ページ |
| 関連実績カード | 実績詳細ページ |

### 設計原則

- カード全体クリックを前提にする
- カード内にリンクを増やしすぎず、主遷移先を1つに絞る
- ホバーやタップで操作対象と分かる状態を持たせる

## CTAリンク設計

### 主要CTAのリンク先候補

お問い合わせ / 料金ページ / 実績一覧 / サービス一覧

### 補助CTAのリンク先候補

関連実績 / サービス詳細 / 料金補足 / 選ばれる理由

### CTA配置原則

- 1セクションにつき主導線は原則1つ
- 補助導線を付ける場合も主従を崩さない
- トップ終盤、サービス詳細終盤、実績詳細終盤、料金ページ終盤では問い合わせを主導線にする

---

## 関連リンク設計

| 種別 | 利用場面 | 役割 |
|:---|:---|:---|
| 関連サービスリンク | 料金ページ / 実績詳細 / 将来の資料ページ | サービス理解補助 |
| 関連実績リンク | サービス詳細 / 実績詳細 | 信頼形成、回遊性向上 |

### 設計原則

関連性の基準を持つ（同カテゴリ / 同業種 / 同支援内容）。後続運用で一貫した紐付けルールを維持する。

## パンくずリンク設計

### 基本構造

| ページ | 階層 |
|:---|:---|
| サービス詳細 | トップ → サービス → 各サービス |
| 実績詳細 | トップ → 実績 → 各実績 |
| 資料詳細（将来） | トップ → 資料 → 各資料 |
| 記事詳細（将来） | トップ → ブログ → 各記事 |

### 設計原則

- 実際のサイト構造と一致させる
- 存在しない階層を作らない
- パンくずと内部リンクの整合を保つ

## ページ内アンカーリンク設計

### 初期方針

- 初期公開では必須にしない
- 長文化が進んだ場合の拡張余地を持つ
- アンカーが増えすぎて複雑化しないことを優先する

---

## 内部リンクの優先順位

| 優先度 | 対象 |
|:---|:---|
| 最優先 | お問い合わせ / サービス詳細 / 実績詳細 / 料金ページ |
| 優先 | サービス一覧 / 実績一覧 / 選ばれる理由 / プライバシーポリシー |
| 後続拡張 | FAQ / 資料ページ / 記事ページ / カテゴリページ / 法務一覧 |

---

## リンク文言設計方針

### 推奨パターン

- サービス内容を見る
- 参考レンジを確認する
- 実績詳細を見る
- このような案件を相談する
- 関連する実績を見る

### 非推奨パターン

- こちら / 詳細はこちら / 読む（リンク先が想像しにくい文言の乱用）

---

## SEO観点での内部リンク方針

- 主要ページに評価を集める
- 一覧ページから詳細ページへ適切に流す
- 詳細ページから関連ページへ自然に返す
- リンク文言を意味あるものにする
- 同一リンク先への重複リンクを増やしすぎない

---

## GEO観点での内部リンク方針

- AIにページ関係が伝わりやすい構造にする
- サービス詳細と実績詳細の関係を整理する
- 価格ページとサービスページの関係を整理する
- 問い合わせが最終行動ページであることを明確にする

---

## 将来拡張時の内部リンク方針

| 追加ページ | リンク先 |
|:---|:---|
| FAQページ | 関連サービス詳細 / 料金 / お問い合わせ |
| 資料ページ | 関連サービス / 関連実績 / お問い合わせ |
| 記事ページ | 関連サービス / 関連実績 / 料金 / お問い合わせ |
| カテゴリページ | 一覧と詳細の中間ハブ。乱立させない |

---

## 内部リンク運用ルール

- 新規ページ追加時は、どこから流入させるか・どこへ送るかを決めてから作る
- 孤立ページを作らない
- 不要なリンクは増やさない
- リンク切れを防ぐ
- リンク先の内容変更時は文言も見直す

---

## 内部リンクで避けること

- 同一ページ内で同じリンク先を何度も乱立させること
- 関連性のないページを無理につなぐこと
- リンク先が未整備のまま文脈だけ作ること
- 問い合わせを全ページで煽りすぎること
- 法務ページを主導線化すること

---

## 内部リンクの成功条件

- ユーザーが次に何を見ればよいか分かる
- サービス、料金、実績、問い合わせの関係が整理されている
- 一覧から詳細へ自然に進める
- 詳細から関連情報へ自然に回遊できる
- 問い合わせが文脈で支えられている
- 検索エンジンと生成AIがページ関係を理解しやすい
- 新規ページ追加時もルールを流用できる
