# SocialBoost 公式Webサイト コピーライティング設計書

---

## 文書目的

本書は、SocialBoost 公式Webサイト全体における文章表現、見出し表現、CTA文言、サービス説明、実績説明、料金説明、問い合わせ導線文言の作成ルールを定義し、全ページで一貫したブランドトーンと営業力を両立させることを目的とする。
本書は、要件定義書、情報設計書、各ページ設計書、SEO設計書、GEO設計書を補完し、後続のコピー作成、実装、改善運用の基準文書とする。

## 前提

本フェーズでは実装を行わない。
本書はコピー設計文書であり、画面実装、文言反映、CMS登録、公開作業、A/Bテスト実施、デプロイは行わない。
ユーザー本人が明示的に実装許可を出すまで、実装フェーズへ進まない。

---

## コピーライティングの役割

SocialBoost のサイトにおけるコピーは、単なる説明文ではない。
役割は、何をしている事業かを即座に理解させ、制作会社との違いを整理し、相談のハードルを下げ、問い合わせへ自然につなげることである。

文章が優先すべきこと：
- 理解しやすいこと
- 信頼できること
- 相談しやすいこと
- 実行力が伝わること

---

## コピーライティングの基本思想

1. 抽象だけで終わらせない
2. 何をしているかがすぐ分かる
3. 制作会社ではなく、事業成長パートナーとして見せる
4. 事業理解、設計力、実行力が伝わる
5. 難しい言葉を使いすぎない
6. 相談のハードルを下げる
7. 営業感を出しすぎず、問い合わせに進める
8. 日本の法人向けサイトとして自然な言葉遣いにする
9. SEOとGEOを意識し、要約しやすい文を作る
10. ページごとに役割の違う文体を使い分ける

## ブランドボイス設計

### 目指す印象

知的 / 整理されている / 誠実 / 実務的 / 先進的 / 落ち着いている / 相談しやすい / 押し付けがましくない

### 避けるべき印象

軽い / ノリだけ / 過度にキラキラした制作会社感 / 意味の薄い横文字だらけ / 上から目線 / 安売り感 / 抽象論だけ / 勢いだけの営業文

| ページ | トーン |

| トップページ | 少し強め、短く印象に残る、抽象と具体をバランスよく混ぜる |

| お問い合わせページ | 安心感重視、やさしい、相談歓迎 |

## コピーライティングの優先順位

1. 何をしている事業かが分かること
2. 何が頼めるかが分かること
3. なぜ頼む価値があるかが分かること
4. 相談してもよさそうだと感じること
5. ブランドとして安っぽく見えないこと

この順序を崩して、表現の美しさだけを優先しない。

## 表現方針

### 推奨する表現

- 事業課題を整理する
- 導線を設計する
- 成果につながる構造を作る
- 必要なものを見極める
- 小さく始めて拡張する
- 制作だけで終わらない
- 運用まで見据える
- 相談しやすい
- 要件整理から対応する
- 分かる形に落とし込む

### 避ける表現

- 世界を変える / 圧倒的 / 唯一無二 / 革命 / 未来を創る
- 最先端だけを連呼する表現
- 意味の薄いカタカナ語の連発
- 抽象的なビジョンだけで終わる表現

---

## 文体ルール

- ですます調を基本とする
- 1文を長くしすぎない
- 1文1メッセージを基本とする
- 接続詞を重ねすぎない
- 専門用語は必要時のみ使い、補足可能な形にする
- 語尾の単調な連続を避ける
- 言い切りが必要な箇所では短く断定する
- 誇大表現を避ける

---

## 見出し設計ルール

### 基本ルール

- 短い、意味が分かる、抽象だけで終わらない
- 1見出し1テーマ
- ページ全体でトーンを揃える

### 推奨パターン

- 事業課題から、必要な打ち手を整理する
- 制作で終わらない、成果につながる導線設計
- Web制作、システム開発、アプリ開発まで一気通貫で支援
- 実績を見ながら、相談のイメージを持てる
- 要件が固まっていなくても、相談できます

### 避けるパターン

- 価値を最大化する / 未来をデザインする / 革新的なソリューション（何のページか分からない表現）

---

## サブコピー設計ルール

- 見出しの意味を補強する
- 抽象を具体に落とす
- 長くしすぎない、見出しの言い換えだけにしない

例：Web制作、Webシステム開発、アプリ開発、AI導入支援まで、事業フェーズに合わせて設計と実行を行います。

---

## 本文設計ルール

- 結論から入る
- 必要なら理由を続ける
- 長い抽象論にしない
- 箇条的に整理できる情報は整理する
- 画像やPDFを見ないと理解できない本文にしない

---

## CTA文言設計

### 主要CTA候補

- お問い合わせする
- 無料相談する
- 相談してみる
- この内容で相談する
- 要件整理から相談する

### 補助CTA候補

- 実績を見る / 料金を見る / サービスを見る / 関連実績を見る / 参考レンジを見る

### CTA文言ルール

- 短い、押した先が想像できる、煽りすぎない、安心感がある

### 避けるCTA文言

- 今すぐ申し込む / 絶対に失敗しない / 最短で成功する（誇張や煽りが強い表現）

## トップページのコピー設計

### ファーストビュー見出しの方向性候補

- 事業課題を整理し、成果につながる導線をつくる
- 制作、開発、AI活用まで一気通貫で支援
- サイトを作るだけで終わらない。事業を前に進める仕組みまで設計する

### サブコピーの方向性候補

Web制作、LP制作、Webシステム開発、アプリ開発、AI導入支援まで、事業フェーズに合わせて設計と実行を行います。

### 課題訴求コピー例

- 何を頼めばいいか、まだ整理できていない
- サイトを作っても、集客や問い合わせにつながらない
- 業務を効率化したいが、何から手を付けるべきか分からない
- アプリやシステムにしたいアイデアはあるが、要件が固まっていない

---

## サービスページのコピー設計

### 一覧ページ見出しの方向性候補

- 事業フェーズに合わせて、必要な支援を選べます
- 制作、開発、改善まで、目的に合わせて整理できます

### サービスカードの一文説明例

| サービス | 説明例 |
|:---|:---|
| Web戦略設計 | 事業課題を整理し、必要な導線や構成を設計します |
| LP制作 | 集客や問い合わせにつながる訴求設計とLP制作を行います |
| Webシステム開発 | 予約管理や顧客管理など、業務に必要な仕組みを形にします |
| アプリ開発 | MVPから会員向けアプリまで、目的に応じて設計と開発を支援します |

### サービス詳細ページの必要要素

誰向けか → 課題 → 提供内容 → 対応範囲 → 進め方 → 関連実績 → 料金導線 → 問い合わせ導線

### 対象顧客の書き方例

- 新規事業の立ち上げに合わせて、最初の導線設計から整理したい方
- 既存サイトを見直し、問い合わせや導線の改善を進めたい方
- 業務を仕組み化したいが、要件整理から相談したい方

### 課題の書き方例

- 何を作るべきかはある程度見えているが、どう構成すれば成果につながるか整理できていない
- 既存サイトはあるが、情報が散らばっていて、問い合わせ導線も弱い
- 手作業が多く、予約や顧客管理を仕組み化したい

---

## 料金ページのコピー設計

### 料金の考え方の書き方例

ご相談内容によって必要な設計範囲や機能が異なるため、料金は参考レンジでご案内しています。
最初から大きく作り込むのではなく、目的に合わせて小さく始める進め方も可能です。

### 変動要因の書き方例

ページ数や機能数だけでなく、情報整理の範囲や更新性、管理画面の有無によっても費用は変わります。

### 避ける書き方

安さだけを強調 / 一律に見せる / 細かすぎる条件の列挙 / 強い値引き訴求

---

## 実績一覧ページのコピー設計

### カード短文のルール

短い、曖昧でない、案件の輪郭が分かる、詳細を読みたくなる

### 例

- 通信制高校向けSEOメディアの提案と導線設計
- 海洋散骨サービスの法人向けサイト設計
- 店舗向け予約導線の改善を前提としたLP制作

---

## 実績詳細ページのコピー設計

### 書き方のルール

- 案件固有性を出す、一覧の文を使い回さない、事実ベースで整理する
- 成果が曖昧なら狙いや設計意図を誠実に書く
- 画像やPDFを見ないと意味が分からない構造にしない

### 各セクションの書き方例

| セクション | 例 |
|:---|:---|
| 案件概要 | 新規事業の立ち上げにあたり、サービス内容の整理とWeb導線の設計を目的に制作した案件です。 |
| 課題 | 情報量が多く、誰に何を伝えたいかが分散していたため、導線整理と訴求の優先順位付けが必要でした。 |
| 対応範囲 | 情報設計 / ページ構成設計 / UI設計 / 制作方針整理 / 提案資料作成 |
| 成果または狙い | 問い合わせ導線を明確にし、初見ユーザーでもサービス内容を理解しやすい構造を目指しました。 |

## 選ばれる理由ページのコピー設計

### 理由見出しの方向性候補

- 事業理解から、必要な打ち手を整理する
- 制作だけで終わらず、改善まで見据える
- 必要なものだけを見極めて提案する
- 小さく始めて、段階的に広げられる

### 説明文のルール

短い、具体的、比較軸になる、主観だけで押し切らない

## お問い合わせページのコピー設計

### 歓迎メッセージ例

要件が固まっていない状態でも、そのままご相談ください。
Web制作、システム開発、アプリ開発、AI導入支援まで、分かる範囲でご記入いただければ問題ありません。

### 送信完了文の例

お問い合わせを受け付けました。
内容を確認のうえ、折り返しご連絡いたします。
必要に応じて、進め方や予算感の整理も含めてご案内します。

### エラー文の例

- メールアドレスの形式をご確認ください
- 相談内容を入力してください

## SEOを意識したコピー設計

- ページごとに役割を変える
- 同じ説明文を使い回さない
- 見出しと本文を一致させる
- キーワードを不自然に詰め込まない
- 一覧ページは短文化、詳細ページは固有内容を持たせる

---

## GEOを意識したコピー設計

- 要約しやすい、1文1意味、見出しだけで意味が分かる
- サービス、課題、対応範囲、成果を明確に分ける
- 重要情報を画像やPDFだけに閉じ込めない

### GEO向け推奨構造

定義文 → 対象文 → 課題文 → 範囲文 → 成果文 → 相談導線文

例：
- このサービスは、新規事業の立ち上げや既存サイトの改善を検討している事業者向けです。
- よくある課題として、情報整理が不十分、導線が弱い、要件が曖昧なまま進んでしまう、などがあります。
- 対応範囲は、企画、情報設計、制作、改善提案までです。

## 禁止表現

| 種別 | 例 |
|:---|:---|
| 誇大表現 | 必ず成果が出る / 絶対に成功する / 業界最安級 / 日本一 / 圧倒的No.1 |
| 曖昧表現 | 価値を最大化 / 本質的な変革 / 未来を創る だけで終わる表現 |
| 安売り表現 | 格安 / 激安 / 今だけ / 破格 |
| 横文字の乱用 | ソリューション / イノベーション / ディスラプト を連続使用 |

## コピーの長さルール

| 対象 | 目安 |
|:---|:---|
| ファーストビュー見出し | 短く強く、1行〜3行程度 |
| セクション見出し | 一読で意味が分かる長さ |

| 本文 | 1段落あたり2文〜4文程度で区切る |

## コピー制作時の確認観点

- 読んだ瞬間に何のページか分かるか
- 制作会社サイトに見えすぎていないか
- 相談のハードルを下げられているか
- 事業理解と実行力が伝わるか
- 難しすぎないか / 安っぽくないか
- 一覧と詳細で役割が分かれているか
- SEOとGEOの両方で要約しやすいか

---

## 運用ルール

- 新規ページ追加時は、先にそのページの役割を定義してから文言を作る
- サービス文言と実績文言を混同しない
- 料金更新時は、文言のトーンも見直す
- 実績追加時は、案件固有性のある文章を必ず書く
- 問い合わせ改善時は、送信率だけでなく不信感が出ていないかも確認する
