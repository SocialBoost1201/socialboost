# SocialBoost 公式Webサイト メタ情報設計書

---

## 文書目的

本書は、SocialBoost 公式Webサイトにおける title、description、OG情報、ページ名、共有時の見え方、検索結果上の要約表示方針を定義し、SEO設計、GEO設計、構造化データ設計、コピーライティング設計を補完することを目的とする。
本書は、後続の実装設計、OG画像設計、運用設計、改善施策の基準文書とする。

## 前提

本フェーズでは実装を行わない。
本書はメタ情報の設計文書であり、metadata実装、head設定、OG画像生成、SNSカード設定、デプロイは行わない。
ユーザー本人が明示的に実装許可を出すまで、実装フェーズへ進まない。

---

## メタ情報設計の基本思想

メタ情報は、検索結果でクリックされるための短い広告文であると同時に、検索エンジン、SNS、生成AIに対して、そのページが何のページかを端的に伝える要約でもある。

重視する点：
- 分かりやすいこと
- ページ内容と一致していること
- ブランド名とページ役割が伝わること
- 誇張しないこと

---

## メタ情報設計の目的

1. 検索結果で、何のページかが一目で分かること
2. ブランド名とページ役割が適切に伝わること
3. 主要ページでクリック判断しやすいこと
4. SNS共有時や外部共有時に、安っぽく見えないこと
5. ページ本文と矛盾しないこと
6. 将来的なページ追加時にも同じルールで拡張できること

---

## 基本方針

1. ページごとに固有の title と description を持たせる
2. タイトルは短く、意味が明確であること
3. description は要約として成立し、クリック後の内容と一致すること
4. ブランド名の表記ルールを統一する
5. 一覧ページと詳細ページで役割が分かるようにする
6. OG情報は共有用として設計し、見栄えと内容一致を重視する
7. 実績詳細やサービス詳細では、固有名詞やページ固有性を反映する

## title 設計方針

### 基本ルール

- ページ名またはページ役割を明確に入れる
- 必要に応じてブランド名を後置する
- 冗長にしない、不自然なキーワード詰め込みをしない
- ページ本文の内容と一致させる

### 推奨構成

`ページ主題 | ブランド名`

---

## description 設計方針

### 基本ルール

- そのページで何が分かるかを書く
- 抽象論だけで終わらない
- 過剰な営業表現を避ける
- ページごとに固有性を持たせる
- 同じ description を流用しない

### 推奨構造

ページの役割 → 対応内容または得られる情報 → 必要なら相談や実績導線の補足

---

## OG情報設計方針

### 対象

`og:title` / `og:description` / `og:image` / `og:type` / `og:url` / 必要に応じて twitter card 系情報

### 基本方針

- title と description は検索用メタと整合させる
- og:image はブランド感とページ役割が伝わるものを使う
- 文字を詰め込みすぎない、ロゴを埋もれさせない
- 青系ブランドと白ベースの清潔感を保つ

## OG画像設計方針

### トップページ用OG画像

ブランド全体の印象を伝える。ロゴ、簡潔なサブコピー、青系ブランド、白ベースを基本とする。

### 下層ページ用OG画像

ページタイプごとにテンプレート化を前提とする。

| テンプレート候補 | 用途 |
|:---|:---|
| サービスページ用 | サービス名＋ブランドロゴ |
| 料金ページ用 | 料金＋ブランドロゴ |
| 実績ページ用 | 実績＋ブランドロゴ |
| お問い合わせページ用 | ご相談＋ブランドロゴ |

### 共通ルール

- 文字を詰め込みすぎない、小さな文字を使わない
- ロゴとページタイトルの主従を明確にする
- 背景は明るく、ブランドカラーを活かす
- SNSで縮小表示されても判別しやすい構成

---

## 個別化方針

### 実績詳細ページ

- 案件ごとに title と description を個別化する
- 案件名を title に含める
- description では案件概要、対応内容、何が見られるかを要約する

### サービス詳細ページ

- サービス名を title の先頭に置く
- description では対象顧客、主な課題、対応範囲、相談可能性を要約する
- 全サービスで同じ description を流用しない

---

## メタ情報の重複防止方針

- トップページとサービス一覧で同じ文を使わない
- サービス一覧とサービス詳細で同じ文を使わない
- 実績一覧と実績詳細で同じ文を使わない
- 料金ページとお問い合わせページで同じ文を使わない
- ブランド説明文を全ページに共通流用しない

---

## 検索結果上で避けるべき表現

| 種別 | 例 |
|:---|:---|
| 誇大表現 | 業界最安 / 絶対 / 圧倒的 / 完全成功 |
| 抽象だけの表現 | 未来を創る / 価値を最大化 / 本質を変える |
| 意味の薄い列挙 | 同じワードの重複列挙 |

---

## GEOとの整合方針

- title と description は、AIがページ概要を把握するための短い定義文としても機能させる
- ページ本文と要約が一致することを優先する
- OG情報も、共有されたときにページ内容が誤解されないことを重視する

---

## 更新運用方針

| 更新頻度 | 対象 |
|:---|:---|
| 高頻度 | 実績詳細の title/description、サービス詳細の description、料金ページの description |
| 中頻度 | トップページの description、選ばれる理由の description、お問い合わせの description |
| 低頻度 | ブランド名表記ルール、OG画像テンプレート骨格、meta設計思想 |

### 運用ルール

- 本文更新時はメタ情報との整合を確認する
- 案件追加時は title と description を個別作成する
- OG画像テンプレートを使い回す場合も、文字情報はページごとに最適化する

---

## 計測前提

将来的に確認すべき観点：

- 検索結果でクリックされているか
- title が長すぎて切れていないか
- description がページ内容とズレていないか
- 実績詳細ページの個別メタが弱すぎないか
- SNS共有時に見た目が崩れていないか
