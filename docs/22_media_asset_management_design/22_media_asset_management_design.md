# SocialBoost 公式Webサイト メディア・アセット管理設計書

---

## 文書目的

本書は、SocialBoost 公式Webサイトで使用するロゴ、favicon、OG画像、実績画像、PDF資料、将来追加される各種メディアの配置方針、命名規則、分類ルール、公開可否管理、差し替え運用、品質管理方針を定義し、後続の実装設計、運用設計、テスト設計、更新作業の基準とすることを目的とする。

---

## 対象プロジェクト

| 項目 | 内容 |
|:---|:---|
| 名称 | SocialBoost 公式Webサイト |
| プロジェクトフォルダ | `socialboost` |

---

## 前提

本フェーズでは実装を行わない。
本書はアセット管理の設計文書であり、ファイル移動、ディレクトリ作成、画像最適化、PDF配置、実装反映、デプロイは行わない。
ユーザー本人が明示的に実装許可を出すまで、実装フェーズへ進まない。

---

## アセット管理設計の基本思想

SocialBoost のサイトでは、ロゴ、ブランド画像、実績画像、PDF資料が信頼形成に直結する。
メディア管理は単なるファイル置き場管理ではなく、以下を重視する：

- 整理されていること
- 探しやすいこと
- 差し替えやすいこと
- 公開可否を誤らないこと
- ブランド品質を崩さないこと

---

## アセット管理の目的

1. 必要なファイルを迷わず配置、参照できること
2. favicon、ロゴ、OG画像、実績画像、PDF資料の役割を明確に分けること
3. 将来的な実績追加、画像追加、PDF追加に耐えること
4. 未公開資料や誤った画像公開を防ぐこと
5. ファイル名を見ただけで用途が分かること
6. 実装時に参照パスが複雑になりすぎないこと
7. ブランド品質と表示品質を安定させること

---

## アセット分類

| No. | 分類 | 役割 |
|:---|:---|:---|
| 1 | favicon関連 | ブラウザタブ、ホーム画面追加、PWA補助 |
| 2 | ブランドロゴ関連 | ヘッダー、フッター、OG、ブランド表現用基幹素材 |
| 3 | OG画像関連 | SNS共有、外部共有時のサムネイル |
| 4 | 実績サムネイル画像 | 一覧ページや関連実績の代表画像 |
| 5 | 実績詳細画像 | 案件詳細ページ内のギャラリー画像 |
| 6 | 実績PDF資料 | 提案資料、構成資料、設計資料、営業資料 |
| 7 | 将来の資料ページ用PDF | ダウンロード資料、チェックリスト等 |
| 8 | 記事用画像 | 将来のSEO記事やコラム用 |
| 9 | 補助アイコン・装飾画像 | 必要最小限 |

---

## 配置方針の基本原則

1. ブラウザが直接参照するものは `public` 直下優先
2. ブランド資産は用途別ディレクトリで整理
3. 実績関連は案件単位で整理可能な構造を持つ
4. 画像とPDFはディレクトリ上も分離できる前提
5. ファイル名は英字、小文字、ハイフン区切り
6. 意味不明な連番だけのファイル名を禁止
7. 未公開管理が必要なものは運用ルールを明確化

## 推奨ディレクトリ構成案

```
public/
  favicon.ico
  favicon.svg
  favicon-96x96.png
  apple-touch-icon.png
  site.webmanifest
  web-app-manifest-192x192.png
  web-app-manifest-512x512.png

  brand/
    socialboost-logo.svg
    socialboost-logo.png
    socialboost-symbol.svg
    socialboost-symbol.png

  og/
    og-default.png
    og-home.png
    og-services.png
    og-pricing.png
    og-works.png
    og-contact.png

  works/
    [work-slug]/
      thumb/
        work-thumb-main.webp
      gallery/
        work-gallery-01.webp
        work-gallery-02.webp
      pdf/
        work-proposal-01.pdf
        work-structure-01.pdf

  documents/
    document-checklist-01.pdf

  blog/
    article-sample-cover.webp
```

※ この構成はあくまで設計方針であり、最終採用構成は技術設計と実装方式に合わせて確定する。

---

## favicon関連アセット管理方針

### 配置方針

`public` 直下に配置する（favicon専用フォルダへ分離しない）。

### 対象ファイル

`favicon.ico` / `favicon.svg` / `favicon-96x96.png` / `apple-touch-icon.png` / `site.webmanifest` / `web-app-manifest-192x192.png` / `web-app-manifest-512x512.png`

### 理由

ブラウザやOSがルート直下を参照しやすく、参照パスが短く、設定ミスが減る。

---

## ブランドロゴ管理方針

### 配置方針

`public/brand/` 配下を基本候補とする。

### 命名ルール

ブランド名＋用途＋形式を含める。

| 推奨 | 禁止 |
|:---|:---|
| `socialboost-logo.svg` | `logo-final.png` |
| `socialboost-logo-dark.svg` | `newlogo2.svg` |
| `socialboost-symbol.svg` | `sb.png` |

### ロゴ運用ルール

- 正式ロゴを基準とし、比率変更禁止、勝手な色変更禁止
- 背景に合わせた派生版は明示的に区別する
- デザインシステム設計と連携する

---

## OG画像管理方針

### 配置方針

`public/og/` 配下を基本候補とする。

### 命名例

`og-default.png` / `og-home.png` / `og-services.png` / `og-pricing.png` / `og-works.png` / `og-contact.png` / `og-work-[slug].png` / `og-service-[slug].png`

### 方針

- 主要固定ページは専用OG画像を持つ
- 実績詳細やサービス詳細は将来的に個別OGを持つ余地を残す
- 初期段階では共通テンプレート運用でもよい

---

## 実績画像管理方針

一覧用サムネイルと詳細用ギャラリー画像を分けて管理する。

### 配置方針

```
works/[work-slug]/thumb/   … サムネイル
works/[work-slug]/gallery/ … ギャラリー画像
```

### 命名ルール

| 種別 | 推奨例 | 禁止例 |
|:---|:---|:---|
| サムネイル | `[slug]-thumb-main.webp` | `img1.png` |
| ギャラリー | `[slug]-gallery-01.webp` | `final2.jpg` |

- 連番は2桁以上を推奨
- 表示順と `sortOrder` を一致させやすくする

---

## 実績PDF管理方針

### 配置方針

`works/[work-slug]/pdf/` を基本候補とする。

### 命名ルール

案件slug＋資料種別＋連番を含める。

| 推奨例 |
|:---|
| `[slug]-proposal-01.pdf` |
| `[slug]-structure-01.pdf` |
| `[slug]-report-01.pdf` |
| `[slug]-wireframe-01.pdf` |

- 何の資料か分かる名称にする
- PDFタイトル表示とファイル名管理は別物として考える

---

## 公開可否管理方針

| 対象 | 方針 |
|:---|:---|
| 案件全体 | 案件単位で公開/非公開を管理 |
| 画像単位 | 案件は公開でも画像単位で非公開があり得る |
| PDF単位 | 案件は公開でも資料単位で非公開があり得る |

### 運用ルール

- クライアント承認のない資料は非公開
- 機密情報を含む資料は非公開
- 画面に個人情報が残る画像は非公開または加工前提
- 公開前に内容確認を行う

---

## ファイル形式方針

| 対象 | 推奨形式 |
|:---|:---|
| ロゴ | SVG 優先、必要に応じて PNG |
| 実績画像 | WebP 優先候補、必要に応じて PNG/JPEG |
| OG画像 | PNG 優先候補 |
| PDF | PDF を維持、編集元は公開領域外で管理 |

---

## 画質管理方針

| 対象 | 基準 |
|:---|:---|
| ロゴ | にじみやぼやけを許容しない |
| 実績サムネイル | 一覧で粗く見えない、縦横比が崩れない |
| 実績詳細画像 | 拡大閲覧に耐える、過剰に重くしない |
| OG画像 | 共有時の視認性、文字が小さすぎない |
| PDF | 閲覧時に読める品質、不要に大容量にしない |

---

## 命名規則の共通原則

- 英字小文字を基本、単語区切りはハイフン
- 日本語ファイル名は避ける、空白を使わない
- `final` / `new` / `ver2` / `latest` などの曖昧語を避ける

### 推奨パターン

```
[brand]-[asset]-[variant].[ext]
[work-slug]-[type]-[seq].[ext]
[page]-og.[ext]
```

### 例

```
socialboost-logo.svg
og-home.png
yokohama-kimchiya-thumb-main.webp
yokohama-kimchiya-gallery-01.webp
yokohama-kimchiya-proposal-01.pdf
```

---

## 差し替え運用方針

### 原則

- 同名上書きか新規ファイル名かを運用で統一する
- キャッシュや参照ズレの防止は実装方針と合わせて決定

### 推奨運用

| 対象 | 方針 |
|:---|:---|
| ロゴ・OG（固定資産） | 同名更新を基本候補 |
| 実績画像・PDF | 必要に応じて連番または版管理 |

### 版管理方針

公開ファイル名に `v2` `v3` を乱用しない。管理領域で版を保持し、公開領域では整理された最新ファイルのみを置く。

---

## メディア最適化方針

- 画像は公開前に表示用途に合わせて最適化する前提
- 一覧用サムネイルと詳細用画像を兼用しすぎない
- OG画像は共有時の見やすさ基準で作る
- PDFは閲覧性を損なわない範囲で容量を抑える
- 元データと公開データは別管理前提

---

## セキュリティと公開管理方針

- `public` 配下に置くものは原則公開可能なものだけ
- 機密性の高い元ファイルは公開領域外で管理
- PDF公開時は内容確認を必須
- 画像公開時は個人情報や管理画面情報の写り込みを確認
- 誤公開しやすいファイル名を避ける

---

## 将来拡張方針

`documents/` / `blog/` / `case-study/` などの追加があっても同ルールを適用する。
用途別ディレクトリと命名ルールを統一し、ブランド資産・案件資産・記事資産を混在させない。

---

## 運用チェックリスト方針

実装前または公開前に確認すべき観点：

- ファイル名がルールに沿っているか
- 配置先が用途と一致しているか
- 公開可否が妥当か
- 画像品質が十分か
- PDF内容に問題がないか
- 参照パスが正しいか
- 一覧用と詳細用の役割が分かれているか

## 本書で確定する事項

| No. | 決定事項 |
|:---|:---|
| 1 | favicon関連は `public` 直下管理を基本とする |
| 2 | ロゴは `public/brand/` 配下管理を基本候補とする |
| 3 | OG画像は `public/og/` 配下管理を基本候補とする |
| 4 | 実績画像と実績PDFは案件slug単位で整理可能な構造を採用する |
| 5 | 画像は一覧用サムネイルと詳細用ギャラリーで役割を分ける |
| 6 | 画像とPDFはディレクトリ上も運用上も分離して管理する |
| 7 | ファイル名は英字小文字、ハイフン区切り、意味のある命名を原則とする |
| 8 | 公開可否は案件単位とファイル単位の両方で管理できる前提を持つ |
| 9 | `public` 配下には公開可能なファイルのみを置く |
| 10 | 現時点では実装に進まない |
