# SocialBoost 公式Webサイト 運用設計書

---

## 文書目的

本書は SocialBoost 公式Webサイトの公開後の運用方針、更新フロー、コンテンツ管理、監視、問い合わせ対応、改善サイクルを定義する文書である。

サイトは公開して終わりではなく、継続的な改善と運用によって価値が高まる。
本書では、安定運用と事業成長の両方を実現するための運用ルールを定義する。

本書は以下の設計書と連携する。
- 要件定義書
- 基本設計書
- 詳細設計書
- UX設計書

- GEO設計書
- フォーム設計書

## 運用設計の基本方針

1. **継続改善**
   サイトは公開後に改善を続ける前提とする。
2. **データに基づく運用**
   アクセス解析を元に改善する。
3. **コンテンツ更新**
   実績やサービス情報は定期更新する。
4. **安定稼働**
   常に正常にサイトが表示される状態を維持する。
5. **セキュリティ維持**
   依存パッケージや環境を定期更新する。

## コンテンツ更新運用

### 更新対象と更新頻度

| 対象ページ | 更新頻度 |
|:---|:---|
| トップページ | 必要に応じて更新 |
| サービスページ | 半年に1回見直し |
| 料金ページ | 変更時 |
| 実績ページ | 案件完了ごと |
| SEOコンテンツ | 月1回以上 |

### 実績更新ルール

実績はSocialBoostの信頼性を高める重要コンテンツである。

**更新内容：**
- プロジェクト名
- カテゴリ
- 概要
- 成果
- 使用技術
- 画像 または PDF

**公開判断：**
- 機密情報・顧客情報を含むものは掲載しない。

## 問い合わせ運用

### 問い合わせ基本フロー
問い合わせ受付 → 内容確認 → 返信 → 案件化判断

### 対応目標と管理

| 項目 | 内容 |
|:---|:---|
| 返信目標時間 | **24時間以内** |
| 管理項目 | 問い合わせ日時、名前、会社名、メール、相談内容、対応状況 |
| 対応状況ステータス | 未対応 / 対応中 / 対応完了 / 案件化 |

### 問い合わせデータに基づく改善
問い合わせ内容を分析し、「よくある相談」「ニーズ」「課題」を抽出し、サービスページのコンテンツ改善へ反映する。

## 監視運用

### サイト監視

| 項目 | 内容 |
|:---|:---|
| 監視対象 | サイト稼働、ページ表示速度、エラー発生状況、フォーム送信機能 |
| 監視ツール | UptimeRobot または BetterStack |
| 監視頻度 | 常時監視 |
| アラート基準 | サイトダウン、エラー増加、応答遅延 |

## 分析・改善運用

### アクセス解析

| 項目 | 内容 |
|:---|:---|
| 利用ツール | Google Analytics、Google Search Console |
| 分析項目 | 訪問数、流入経路、ページ閲覧数、滞在時間、問い合わせ率 |
| 分析頻度 | 月1回 |

### SEO / GEO運用

**定期確認項目（SEO）：**
- 検索順位
- 流入キーワード
- クリック率
- ページ表示速度

**SEO改善施策：**
- タイトル改善
- 内部リンク改善
- コンテンツ追加

**GEO運用（AI検索最適化）：**
- サイト構造の整理
- FAQ追加
- 専門性の明確化

## システム・技術運用

### 依存ライブラリ更新
- Next.js、Node.js、その他依存パッケージの更新
- **更新頻度：** 3ヶ月に1回

### デプロイ運用
デプロイはCI/CDで行い、**直接本番環境を変更しない**（Git管理で更新する）ルールとする。

**デプロイフロー：**
GitHub push → Vercel build → Preview → Production

### バックアップ運用

| 項目 | 内容 |
|:---|:---|
| 対象 | コード、問い合わせデータ |
| バックアップ方法 | GitHub、クラウドデータストア |
| 頻度 | 毎日 |

---

## 障害対応運用

### 障害対応フロー
障害検知 → 原因調査 → 復旧 → 再発防止策の策定

### 障害レベル定義

| レベル | 状態定義 |
|:---|:---|
| **軽度** | 一部ページエラー、表示崩れ |
| **中度** | フォーム不具合等の主要機能エラー |
| **重大** | サイト停止 |

---

## 改善サイクルと品質チェック

### 月次レビュー（改善サイクル）
以下の分析結果を元に月次でレビューを行う。
- アクセス分析
- 問い合わせ分析
- SEO分析

**具体的な改善内容：**
- UI改善
- コンテンツ追加
- 導線改善

### 定期品質チェック
- **チェック頻度：** 月1回
- **チェック項目：** リンク切れ、表示崩れ、モバイル表示、フォーム送信テスト
