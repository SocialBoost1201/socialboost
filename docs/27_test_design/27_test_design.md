# SocialBoost 公式Webサイト テスト設計書

---

## 文書目的

本書は SocialBoost 公式Webサイトの品質保証のためのテスト方針、テスト範囲、テスト項目、合格基準、実施プロセスを定義する。
本サイトは事業サイトであり、信頼性、表示速度、SEO、ユーザー体験、セキュリティのすべてが一定水準以上である必要がある。
そのため公開前に体系的な品質検証を行う。

本書は以下の設計書と連携する。
- 要件定義書
- 基本設計書
- 詳細設計書
- UX設計書
- SEO設計書
- GEO設計書
- フォーム設計書
- セキュリティ設計書
- インフラ設計書
- 運用設計書

## テスト設計の基本方針

1. ユーザー体験を最優先に検証する
2. すべてのページの表示品質を確認する
3. 導線が自然かを検証する
4. 検索エンジンに正しく認識されるかを確認する
5. モバイル環境で問題なく利用できることを確認する
6. 公開後の運用を想定した安定性を確保する

## テスト詳細項目

### 1. 機能確認テスト
各ページが正常に表示されることを確認する。

**確認内容：**
- ページがエラーなく表示される
- リンク先が正しい
- ページ遷移が正常
- 画像が表示される

### 2. 画面表示テスト
デザイン崩れの有無を確認する。

**確認項目：** レイアウト、余白、フォント、画像サイズ、CTA配置
**ブラウザテスト：** Chrome、Safari、Edge、Firefox
**画面サイズテスト：** 1920px、1440px、1024px、768px、390px

### 3. ユーザー導線テスト
以下の導線を検証する。
`トップページ → サービス → 実績 → お問い合わせ`

**確認項目：**
- 迷わず問い合わせまで到達できる
- 重要情報が見つけやすい
- CTAが視認しやすい

### 4. フォーム動作テスト
**対象ページ：** お問い合わせページ

**確認項目：**
- 必須項目チェック
- メール形式チェック
- エラー表示（ユーザーに分かりやすい、適切な表現であること）
- 送信成功表示

**エラー表示テスト対象：**
担当者名未入力 / メール未入力 / 相談内容未入力 / メール形式エラー

### 5. リンクテスト
すべてのリンク（内部リンク、外部リンク、CTAリンク）を確認する。

**確認内容：**
- リンク切れなし
- 404ページ発生なし

### 6. メディアテスト
**画像テスト確認項目：**
- 画像読み込み、画質、サイズ、表示速度が正常であること
**PDFテスト確認項目（実績ページ）：**
- PDF表示、ダウンロード、リンク動作が正常であること

### 7. レスポンシブテスト
モバイルでの表示確認を行う。

**確認項目：**
- タップ操作、スクロール、文字サイズ、CTAボタンの押しやすさ

### 8. SEO / GEO構造テスト
**SEO確認項目：**
- `title`タグ、`meta description`、見出し構造、内部リンク
- Google検索での表示確認
**GEO確認項目：**
- AI検索エンジンに対する理解度（ページ構造、FAQ、専門性）
**OGP表示テスト：**
- LINE、X、FacebookなどSNSでの表示確認

### 9. パフォーマンステスト
ページ速度を測定する。
**測定ツール：** PageSpeed Insights、Lighthouse

**目標指標：**
- LCP：2.5秒以内
- CLS：0.1以下
- TTFB：500ms以内

### 10. アクセシビリティ確認
- 文字の読みやすさ
- コントラスト
- 操作性

### 11. エラー監視確認
以下のログを確認する。
- エラーログ
- 404エラー
- フォームエラー

## 品質保証プロセス

### 合格基準
- すべてのページが正常表示される
- フォームが正常動作する
- リンク切れがない
- モバイル表示に問題がない
- SEO設定が適切である

### テスト記録
以下の内容を記録領域に残す。
- テスト日、担当者、テスト対象、結果、問題点

### 問題発生時の対応フロー
問題確認 → 原因調査 → 修正 → 再テスト
