# SocialBoost 公式Webサイト 情報設計書（Information Architecture Design）

---

## 文書目的

本書は SocialBoost 公式Webサイトの情報構造を定義する文書である。
ユーザーが必要な情報に迷わず到達できるサイト構造を設計し、SEOおよびAI検索（GEO）にも理解されやすいサイト構造を構築することを目的とする。

本書ではページ階層、ナビゲーション構造、ユーザー導線、コンテンツ配置ルールを定義する。

本書は以下の設計書と連携する。
- 要件定義書
- 基本設計書
- 詳細設計書
- UX設計書
- SEO戦略設計書
- GEO戦略設計書
- CMS設計書
- コンテンツ制作ガイドライン

## 情報設計の基本方針

1. ユーザーが迷わない構造にする
2. 情報をシンプルに整理する
3. 主要情報はトップページから到達可能にする
4. SEOとAI検索（GEO）に理解されやすい構造にする
5. コンテンツの拡張性を確保する

---

## サイト構造とナビゲーション設計

### グローバルナビゲーション
ヘッダーには以下のナビゲーション（4項目）を配置する。
- サービス
- 料金
- 実績
- お問い合わせ

**ナビゲーションの目的：**
- ユーザーが主要情報へ最短で到達する
- 問い合わせ導線を明確にする
- サイト構造をシンプルにする

### サイト全体構造（ページ階層）
サイトは以下の3階層を基本とする。

- **Level 1**：トップページ
- **Level 2**：サービス / 料金 / 実績一覧 / お問い合わせ
- **Level 3**：実績詳細

### URL設計

| ページ名 | URLパス |
|:---|:---|
| トップページ | `/` |
| サービス | `/services` |
| 料金 | `/pricing` |
| 実績 | `/works` |
| 実績詳細 | `/works/[project-name]` |
| お問い合わせ | `/contact` |

## ユーザー導線と内部リンク設計

### ユーザー導線設計
**基本フロー：**
`トップページ  →  サービスページ  →  実績ページ  →  お問い合わせページ`

**目的：** サービス理解 → 信頼構築 → 問い合わせ誘導

### 内部リンク設計
- `トップページ → サービス`
- `サービス → 実績`
- `実績 → お問い合わせ`

### CTA配置
各ページ（トップ、サービス、実績）の最下部および適切な箇所に、問い合わせ誘導を目的としたCTAを配置する。

## モバイル・SEO・GEO情報設計

### モバイル情報設計
スマートフォンでは以下を重視する。
- スクロール導線の分かりやすさ
- タップしやすいCTAボタン
- 読みやすいレイアウト（適切な余白とフォントサイズ）

### SEO情報設計
検索エンジンが理解しやすいよう、見出し構造を明確にする。
- `H1`：ページテーマ
- `H2`：主要コンテンツ
- `H3`：詳細説明

### GEO（AI検索）情報設計
AI検索エンジン（Generative AI）が内容を理解・抽出しやすい構造とする。
- 明確なサービス説明
- 結論・課題・成果がまとまった実績説明
- QA形式を用いたFAQコンテンツ

---

## 拡張性設計

将来的に以下のページあるいはコンテンツ群の追加を想定し、ナビゲーションやルーティングが破綻しない構造としておく。
- ブログ（`/blog`）
- SEO記事（`/articles` 等）
- AI記事・コラム

## 本書で確定する事項

| No. | 決定事項 |
|:---|:---|
| 1 | グローバルナビゲーション（ヘッダー）は4項目（サービス,料金,実績,お問い合わせ）で構成する |
| 2 | サイト構造はLevel3（実績詳細）までのシンプルな階層とする |
| 3 | 基本となるユーザー導線（トップ→サービス→実績→問い合わせ）を設け、内部リンクを最適化する |
| 4 | SEO（見出し階層 H1-H3）およびGEO（FAQや明確な説明）の構造を意識する |
| 5 | モバイルでの閲覧（スクロール導線、CTAのタップ性）を最前提とする |
| 6 | 将来的なブログ等（CMS連携など）のページ拡張を想定する |
| 7 | 現時点ではフロントエンド実装やルーティング処理などは行わない |
