# SocialBoost 公式Webサイト UIデザインガイドライン

---

## 文書目的

本書は SocialBoost 公式WebサイトのUIデザインのルールと設計思想を定義する文書である。
デザインの統一性、視認性、操作性、ブランドイメージを維持することを目的とする。

本サイトは「最先端」「プロフェッショナル」「信頼性」を感じさせるデザインを基本とする。
日本のハイレベルな制作会社サイトのUI品質を参考にしつつ、過度な装飾ではなく洗練されたUIを採用する。

**参考サイト（デザインベンチマーク）：**
- `yucca.jp`
- `feel-design.jp`
- `dotzero.jp`

本書は以下の設計書と連携する。
- 要件定義書
- 基本設計書
- 詳細設計書
- UX設計書
- 情報設計書
- SEO戦略設計書
- GEO戦略設計書

## UI設計の基本思想とコンセプト

### 基本思想
1. シンプルで洗練されたデザイン
2. 余白を活かしたレイアウト
3. 読みやすいタイポグラフィ
4. 視線誘導を意識した構成
5. ブランドロゴが映える配色

### デザインコンセプト
- 先進的 (Advanced)
- プロフェッショナル (Professional)
- クリーン (Clean)
- モダン (Modern)

## トーン＆マナー設定

### カラーパレットと役割
| 役割 | カラー指定方針 |
|:---|:---|
| **ベースカラー** | トップページや共通背景には「ダーク背景」を採用。コンテンツエリアによりダークまたは白背景を併用。 |
| **アクセントカラー** | ロゴカラーを採用。（CTA等、最も目を引く要素に限定使用） |
| **テキストカラー** | ダークベース：ホワイト および ライトグレー / 白ベース：ダークグレー等 |

### タイポグラフィ
フォントは読みやすさを重視し、モダンで洗練されたゴシック体等を採用する。
- **見出し：** 太字、大きめのサイズでメリハリをつける
- **本文：** 中サイズ、視認性を高めるための適切な行間と字間を確保

**見出し階層のデザインルール：**
- `H1`：ページタイトル（最も大きく重みのある装飾）
- `H2`：セクションタイトル（視線を区切る明確なデザイン）
- `H3`：サブタイトル（本文へのシームレスな誘導）

## UIコンポーネント設計

### ボタンデザイン
ボタンはシンプルで視認性の高いものを使用する。
**CTAボタンの要件：**
- やや丸みを持たせた角丸（親しみやすさと押しやすさ）
- アクセントカラー（ロゴカラー）の単色塗りまたはグラデーション
- ホバー時に控えめなアニメーション（シャドウの増加、わずかな透過等）

**ボタン例：**
- `お問い合わせ`
- `サービスを見る`

### カードデザイン（実績表示等）
実績表示等は情報を整理しやすい「カードUI」を使用する。
**カード構成要素：** 画像、タイトル、概要
**ホバーアクション：** 画像の軽い拡縮（スケール）、カード全体へのソフトなシャドウ追加

### アイコン
過度に主張しない「シンプルな線アイコン（Line Icon）」を採用する。
- **主な用途：** サービス説明、特徴・メリットの説明用

## レイアウトと余白設計

### レイアウト設計
整然とした「グリッドレイアウト」を採用する。
- 12カラム または よりシンプルなグリッドシステム

### 余白（マージン/パディング）設計
**広めの余白（ホワイトスペース）**を意識的に確保する。
- **理由：** 高級感（洗練さ）の演出、および可読性の向上

---

## 画像とアニメーション設計

### 画像デザインとルール
- 高品質な画像を使用する（実績画像、UIイメージ画像等）
- 過度な装飾（強すぎるシャドウや派手な枠線）は避ける
- 極力、自然光やリアルなビジネスシーンを感じさせる「自然な写真」を選定する

### アニメーション設計
派手で過度なアニメーションは使用せず、UX向上と視線誘導を目的とした「マイクロインタラクション」を主とする。
**使用例：**
- スクロールフェードイン（要素がふわっと現れる）
- ボタンやリンクのホバーエフェクト
- カードUIホバー時の自然なアニメーション

## 共通セクション設計

### ヘッダー（グローバルナビゲーション）
**固定ヘッダー（スクロール追従型）**
- **配置メニュー：** サービス、料金、実績、お問い合わせ

### フッター
**シンプル構成**
- コピーライト、必要な主要リンクのみを配置し、ごちゃごちゃさせない。

---

## マルチデバイス・アクセシビリティ対応

### モバイルUI（レスポンシブ設計）
スマートフォン（モバイル）利用を最適化する。
- 押しやすい大きめのタップターゲット（ボタン、リンク）
- モバイルでも視認性が高い文字サイズ
- 縦スクロールに最適化したシームレスなコンテンツ設計
- **対応ブレイクポイント：** PC、タブレット、スマートフォン

### アクセシビリティ
- 十分な文字コントラストの確保（WCAGガイドライン等に準拠）
- 読みやすいフォントの選定
- どのデバイスでも直感的に操作できるUI設計

### ブランド表現の徹底
- ページ全体を通して `SocialBoost` のロゴが目立ち、ブランドを認知させるデザイン。
- キーカラーの統一、余白計算による整然とした美しさの表現。

## 本書で確定する事項

| No. | 決定事項 |
|:---|:---|
| 1 | 参考サイト（yucca.jpなど）に準じた、シンプルで洗練された高級感あるUIを採用する |
| 2 | ダークベースを基調としたデザインとし、ブランドロゴカラーをアクセントにする |
| 3 | 実績等はカードUIで整理し、ホバー等のアニメーションは洗練された控えめなものにする |
| 4 | CTAボタンを明確にし、視線誘導と問い合わせへの導線を強化する |
| 5 | 広めの余白をとり、グリッドレイアウトでプロフェッショナルな印象を与える |
| 6 | モバイルファーストでレスポンシブ設計・UI最適化を行う |
| 7 | 過度なアニメーションは避け、心地よいUXに寄与する動きのみを採用する |
| 8 | 現時点ではフロントエンド実装・コーディングは行わない |
