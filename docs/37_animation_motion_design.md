# SocialBoost 公式Webサイト アニメーション・モーション設計書

---

## 文書目的

本書は SocialBoost 公式Webサイトのアニメーションおよびモーションデザインの設計方針を定義する。
サイトの体験価値を高めるために、視線誘導、情報理解、ブランドイメージ向上を目的としたモーションを設計する。

本サイトのアニメーションは「派手さ」ではなく「洗練」「滑らかさ」「先進性」を重視する。

本書は以下の設計書と連携する。
- UIデザインガイドライン
- UX設計書
- 情報設計書
- 詳細設計書

## モーション設計の基本方針

1. 過度な演出（Too much animation）は行わない
2. UXを向上させる目的でのみ使用する
3. スクロール体験を滑らかにする
4. ユーザーの視線誘導を助ける
5. サイト全体で統一された動きにする

**参考サイト（モーションベンチマーク）：**
- `yucca.jp`
- `feel-design.jp`
- `dotzero.jp`

## モーション要素設計

### 1. スクロールアニメーション
ページスクロール時にコンテンツを自然に表示する。
- **適用箇所：** サービス紹介、実績セクション、各種コンテンツブロック
- **アニメーション例：**
  - **フェードイン：** 自然な表示、読みやすさ向上が目的（テキスト、画像表示）
  - **スライドイン：** 下から、または横からの表示（コンテンツセクションの表示）
  - **ズームイン：** 特定要素の強調

### 2. カードアニメーション
実績カード等のインターフェース要素における挙動。
- **ホバー時アクション：** 画像の軽い拡大（スケールアップ）、影（シャドウ）の表示
- **目的：** インタラクションの強化とクリックできることの明確化

### 3. CTAアニメーション
問い合わせや詳細ページへ誘導するボタンの挙動。
- **ホバー時アクション：** 背景色等の色変化、ボタン全体の軽い拡大
- **目的：** ユーザーのクリック誘導

### 4. ヘッダーアニメーション
- **スクロール時アクション：** ヘッダーの固定表示（Sticky）、背景色や透過度の変更
- **目的：** スクロール時の本文への没入感向上と、ナビゲーションの視認性確保

### 5. 画像・ヒーローアニメーション
トップページのヒーローセクション等での演出。
- **アクション：** ゆるやかな動き、パララックス（視差）効果
- **目的：** サイト訪問直後の視線誘導と、先進的・プロフェッショナルなイメージの醸成

## スクロール・ページトランジション

### スムーズスクロール
ページ内遷移や全体的なスクロールにスムーズスクロール（Lenis等）を採用する。
- **目的：** 高級感の演出と、快適なブラウジング操作感の提供

### ページトランジション
ページ遷移時（トップ、サービス、実績間等）のシームレスなアニメーションを検討する。
- **適用例：** フェードトランジション（画面遷移時の白飛びを防ぐ）

## モバイル・パフォーマンス・アクセシビリティ

### モバイルモーション設計
スマートフォン端末では、PCに比べてアニメーションを「軽量化」または「削減」する。
- **理由：** デバイスのパフォーマンス維持、バッテリー消費の抑制、スクロール阻害の防止

### アニメーション速度（Duration/Easing）
心地よい操作感のために、以下の速度を推奨する。
- `200ms`（高速なUIフィードバック：ボタンホバー等）
- `300ms`（標準的な要素表示等）
- `500ms`（ゆったりとした大きなセクション表示等）
- ※ユーザー操作への反応（ホバー等）は高速に設定し、もたつきを感じさせない。

### パフォーマンス配慮
アニメーションが描画パフォーマンスやページ読み込み速度（Core Web Vitals等）に悪影響を与えないよう設計する。
- **対策：**
  - `transform` や `opacity` など GPU を活用できる軽量なアニメーションプロパティに限定
  - 画面外の過剰なアニメーション処理の回避（Intersection Observer 等の活用）

### アクセシビリティ（A11y）配慮
OSの設定で「視差効果を減らす（`prefers-reduced-motion`）」を有効にしているユーザーに対し、アニメーションを無効化またはフェード等に軽減する設計を標準とする。

## 本書で確定する事項

| No. | 決定事項 |
|:---|:---|
| 1 | スクロールアニメーション（フェードイン/スライドイン等）を使用しコンテンツを自然に表示する |
| 2 | カードホバーやCTAボタンには、視認性を高めるインタラクション効果（軽い拡縮・色変化）を付ける |
| 3 | ヘッダー領域はスクロール時に固定表示（背景変化）とする |
| 4 | サイト全体でスムーズスクロールを採用し、高級感と滑らかさを演出する |
| 5 | モバイル環境においては軽量化を優先する |
| 6 | 描画パフォーマンスを優先し、GPUを効果的に用いた軽量アニメーションに限定する |
| 7 | アクセシビリティに配慮し視差効果軽減の設定に対応する |
| 8 | 現時点では実装（コーディング）は行わない |
