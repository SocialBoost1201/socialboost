## 文書目的

本書は SocialBoost 公式Webサイトのフロントエンド構造、技術構成、ディレクトリ設計、コンポーネント設計ルールを定義する。
本サイトは単なるコーポレートサイトではなく、将来的にWebシステム開発・アプリ開発・AIサービスなどの実績や機能を拡張していく前提のため、拡張性と保守性を重視したアーキテクチャを採用する。

本設計は以下を実現することを目的とする。
- 拡張性の高い構造
- 再利用可能なコンポーネント設計
- SEOに強いレンダリング
- 高速表示
- モダンなUI実装

本書は以下の設計書と連携する。
- 要件定義書

- 詳細設計書

## ディレクトリ構成

プロジェクトのルートレベルにおけるフロントエンドのディレクトリ構成は以下とする。

```text
socialboost/
├── app/          # App Router (ページおよびルーティング)
├── components/   # 共通・再利用可能なUIコンポーネント
├── features/     # ページ単位・機能単位のコンポーネント
├── lib/          # 共通処理、ユーティリティ、設定
├── styles/       # グローバルスタイル管理
├── public/       # 静的アセット (画像、フォント等)
└── docs/         # ドキュメント管理
```

### 各ディレクトリの詳細

#### `app` ディレクトリ (Next.js App Router)
```text
app/
├── page.tsx
├── services/page.tsx
├── pricing/page.tsx
├── works/page.tsx
└── contact/page.tsx
```

#### `components` ディレクトリ
再利用可能なUIコンポーネントを配置する。
- `Header`
- `Footer`
- `CTAButton`
- `SectionTitle`
- `Container`
- `ProjectCard`

#### `features` ディレクトリ
ページ単位の機能コンポーネントを管理し、責務を分離する。
- `services`
- `works`
- `contact-form`

#### `lib` ディレクトリ
共通処理や設定を配置する。
- `constants`
- `utils`
- `config`

#### `styles` ディレクトリ
グローバルスタイル領域。
- `globals.css`

#### `public` ディレクトリ
静的アセット領域。
- `images/`: 実績画像、サービス画像、OGP画像等のassets設計領域
- `icons/`
- `favicon`
- `logo`

## コンポーネント設計原則と分類

### 設計原則
- **単一責任：** 1つのコンポーネントは1つの責務・機能のみを持つ
- **再利用可能：** 特定のコンテキストに固執せず汎用設計とする
- **UIとロジックを分離：** 状態管理やAPI通信等はUI外（hooksやfeatures）で扱う

### コンポーネント分類
| 分類 | 内容とコンポーネント例 |
|:---|:---|
| **UIコンポーネント** | 汎用的で状態を持たない純粋なUI部品（`Button`, `Card`, `Section`, `Container`） |
| **レイアウトコンポーネント** | 画面の骨組みを形成する部品（`Header`, `Footer`, `PageLayout`） |
| **機能コンポーネント** | 特定ページのビジネスロジックや状態を持つ複合部品（`ContactForm`, `ProjectGallery`, `ServiceList`） |

## スタイル・パフォーマンス設計

### スタイル設計（TailwindCSS）
- **理由：** 高速開発、容易なレスポンシブ対応、デザインルールの統一
- **CSS方針：**
  - グローバルCSSを最小化する
  - ユーティリティクラス（Utility-First）を原則として使用する

### レスポンシブ設計
モバイルファーストを採用する。
- **ブレークポイント基準：** `sm`, `md`, `lg`, `xl`

### パフォーマンス最適化と画像最適化
- **Next.js Image Optimization：** 画像圧縮、レスポンシブ画像、高速表示
- **その他最適化施策：** Dynamic Import、Lazy Loading を適切に適用
