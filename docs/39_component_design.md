# SocialBoost 公式Webサイト コンポーネント設計書

---

## 文書目的

本書は SocialBoost 公式Webサイトで使用するUIコンポーネントの設計方針、構造、再利用ルールを定義する。
コンポーネントベース開発を前提とし、再利用性・保守性・拡張性を高めることを目的とする。

本設計は以下を実現する。
- UIの統一
- 開発効率の向上
- コードの再利用
- 将来機能追加への対応

本書は以下の設計書と連携する。
- フロントエンドアーキテクチャ設計書
- UIデザインガイドライン
- モーション設計書
- 情報設計書
- 詳細設計書

## コンポーネント設計方針

1. コンポーネントは再利用可能な構造とする
2. UIとロジックを分離する
3. コンポーネントの責務を明確にする
4. 小さな単位で設計する
5. デザイン統一を保つ

---

## コンポーネント分類

| 分類 | 目的 |
|:---|:---|
| **UIコンポーネント** | UIパーツの共通化 |
| **レイアウトコンポーネント** | ページ構造の共通化 |
| **機能コンポーネント** | 特定機能の実装 |
| **ページコンポーネント** | ページ構造の組み立て |

---

## UIコンポーネント

汎用的で状態を持たない純粋なUI部品。

### Button
- **用途:** CTAボタン、フォーム送信ボタン
- **種類:** Primary / Secondary
- **デザイン:** 角丸、アクセントカラー、ホバーアニメーション

### Card
- **用途:** 実績カード、サービスカード
- **構成:** 画像、タイトル、概要

### Badge
- **用途:** カテゴリ表示、タグ表示

### その他
- `Icon`：アイコン表示
- `SectionTitle`：セクション見出し
- `Container`：コンテンツ幅の制御

---

## レイアウトコンポーネント

画面の骨格を形成する共通部品。

### Header
- **役割:** サイトナビゲーション
- **ナビゲーション項目:** サービス / 料金 / 実績 / お問い合わせ

### Footer
- **役割:** サイト下部情報
- **表示内容:** コピーライト、リンク

### PageLayout
- **役割:** ページ共通レイアウト
- **構成:** Header → Main → Footer

---

## 機能コンポーネント

特定のビジネスロジックや状態を持つ複合部品。

### ContactForm
- **機能:** フォーム入力、バリデーション、送信処理

### ProjectGallery
- **用途:** 実績一覧表示
- **構造:** カードグリッド

### ServiceList
- **用途:** サービス一覧表示

---

## ページコンポーネント

各ページの構造を組み立てるコンポーネント。

| ページ | コンポーネント例 |
|:---|:---|
| **トップページ** (`HomePage`) | `HeroSection` → `ServiceSection` → `WorksSection` → `CTASection` |
| **サービスページ** (`ServicesPage`) | `ServiceOverview` → `ServiceDetails` → `CTASection` |
| **料金ページ** (`PricingPage`) | 料金概要 → プラン → CTA |
| **実績ページ** (`WorksPage`) | `ProjectGrid` → `ProjectCard` |
| **問い合わせページ** (`ContactPage`) | `ContactDescription` → `ContactForm` |

---

## 命名ルール・ディレクトリ構造

### 命名ルール
- **PascalCase** を使用する（例：`ProjectCard`, `ContactForm`, `ServiceSection`）

### ディレクトリ構造
```text
components/
├── ui/          # Button, Card, Badge, Icon
├── layout/      # Header, Footer, PageLayout
└── sections/    # HeroSection, ServiceSection, WorksSection, CTASection
```

---

## レスポンシブ・アクセシビリティ・パフォーマンス

### レスポンシブ対応
TailwindCSSのブレークポイント（`sm`, `md`, `lg`, `xl`）を使用する。

### アクセシビリティ
- ボタンには適切な `aria` 属性を付ける
- キーボード操作に対応する

### パフォーマンス設計
- 不要な再レンダリングを防ぐ（`React.memo` 等）
- 適切なコンポーネント分割を行う
