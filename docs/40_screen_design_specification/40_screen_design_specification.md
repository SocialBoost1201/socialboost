# SocialBoost 公式Webサイト 画面設計書

---

## 文書目的

本書は SocialBoost 公式Webサイトの各画面の構成、レイアウト、表示要素、ユーザー導線を定義する。
デザイン実装前に画面構造を明確にすることで、UI設計、フロントエンド実装、SEO設計の整合性を確保する。

本書では以下を定義する。
- 各ページの画面構成
- UI要素
- CTA配置
- ユーザー導線

本書は以下の設計書と連携する。
- 要件定義書
- 基本設計書
- 詳細設計書
- 情報設計書
- UIデザインガイドライン

- モーション設計書

## 各ページの画面設計

### 1. トップページ画面設計

**目的：** サービス概要理解、信頼構築、問い合わせ誘導
**構成：** ヘッダー → ヒーローセクション → サービス紹介 → 実績紹介 → CTAセクション → フッター

#### ヒーローセクション
- **役割：** 第一印象を決定する
- **要素：** キャッチコピー、サブコピー、CTAボタン
- **CTA例：** `サービスを見る`、`お問い合わせ`

#### サービス紹介セクション
- **役割：** 提供サービスの概要説明
- **表示内容：** Web制作、LP制作、Webシステム開発、アプリ開発、AI導入支援
- **表示形式：** カードUI

#### 実績紹介セクション
- **役割：** 信頼性の提示
- **表示内容：** プロジェクトカード（画像、タイトル、概要）

#### CTAセクション
- **役割：** 問い合わせへの最終誘導
- **表示内容：** お問い合わせボタン

### 2. サービスページ画面設計

**目的：** 各サービス内容を詳細に説明する
**構成：** サービス概要 → 提供内容 → 導入メリット → CTA

**サービス例：**
- Web制作
- LP制作
- Webシステム開発
- アプリ開発
- AI導入支援

### 3. 料金ページ画面設計

**目的：** 料金情報を明確にする
**構成：** 料金概要 → サービス料金 → オプション料金 → CTA

### 4. 実績一覧ページ画面設計

**目的：** 制作実績を紹介する
**表示形式：** カードグリッド
**カード内容：** 画像、タイトル、概要
**インタラクション：** クリックで実績詳細ページへ遷移

### 5. 実績詳細ページ画面設計

**目的：** 実績の詳細紹介
**構成：**
- プロジェクト概要
- 課題
- 実施内容
- 成果
- 画像またはPDF

### 6. お問い合わせページ画面設計

**目的：** 問い合わせ受付
**構成：** 説明文 → 問い合わせフォーム → 送信ボタン
**フォーム項目：**
- 会社名または屋号
- 担当者名
- メールアドレス
- 電話番号
- 相談内容

## CTA配置と各ページの導線

### CTA配置位置

| ページ | 配置箇所 |
|:---|:---|
| **トップページ** | ヒーローセクション、実績セクション後、CTAセクション |
| **サービスページ** | ページ下部（各コンテンツの終了後） |
| **実績ページ** | ページ下部（一覧や詳細の閲覧後） |

### ユーザー導線設計
全画面を通じて以下の基本導線を意識してCTAを配置する。
`トップページ  →  サービス  →  実績  →  お問い合わせ`

## デバイス対応とアクセシビリティ

### レスポンシブ設計
PC、タブレット、スマートフォンの3ブレークポイントに対応する。

### モバイルUIの最適化
スマートフォン画面では以下のUIを優先する。
- タップしやすい大きめのボタン
- 縦スクロールに最適化したコンテンツ設計レイアウト
- 読みやすい文字サイズ

### アクセシビリティの確保
- 読みやすいフォントの選定
- 十分なコントラスト比（文字色と背景色）
- 操作しやすい（大きさと識別性のある）ボタン

## 本書で確定する事項

| No. | 決定事項 |
|:---|:---|
| 1 | トップページは複数セクション構造とし、情報と導線を網羅的に配置する |
| 2 | サービス一覧はカードUIで表示し、直感的な閲覧を促す |
| 3 | 実績はカードグリッドレイアウトとする |
| 4 | CTA（問い合わせ誘導）を各ページの適切な位置（特に下部）に配置する |
| 5 | `トップ → サービス → 実績 → 問い合わせ` の導線を明確にする |
| 6 | モバイルでの閲覧体験を重視したレスポンシブ対応を前提とする |
| 7 | 現時点では実装（コーディングやモック作成）は行わない |
