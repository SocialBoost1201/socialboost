# SocialBoost 公式Webサイト バックエンドアーキテクチャ設計書

---

## 文書目的

本書は SocialBoost 公式Webサイトのバックエンド構造、データ処理、API連携、問い合わせ管理などのサーバー側設計を定義する。

本サイトは主にコーポレートサイトであるが、将来的に以下の拡張を想定している。
- 問い合わせ管理
- CMS化
- 実績管理
- 資料管理
- AIサービス連携

そのため、拡張可能なバックエンドアーキテクチャを前提とする。

## 採用技術とアーキテクチャ方針

### バックエンド構成の基本方針
**サーバーレス構成（Serverless）** を採用する。
- **理由:** 高速、低コスト、スケーラブルであり、運用負荷が低いため。

### 採用インフラ・技術

| 領域 | 採用技術 |
|:---|:---|
| **サーバー（API）** | Next.js Server Functions (Server Actions / Route Handlers) |
| **データベース** | Supabase (PostgreSQL) |
| **認証** | Supabase Auth |
| **ストレージ** | Supabase Storage |

---

## システム構成とバックエンド役割

### システム構成図
```text
ユーザー
  ↓
Next.js フロントエンド
  ↓
Next.js Server Functions (API / Server Actions)
  ↓
Supabase (DB / Auth / Storage)
```

### バックエンドの主要な役割
- **問い合わせ処理:** お問い合わせフォームからのデータ受信および送信処理
- **実績管理:** 将来のCMS化を見据えた実績（プロジェクト）データの保存・取得
- **資料管理:** PDF等のアセット保存・配信
- **将来的機能:** CMS（管理画面）、ユーザー認証・管理

## API設計方針

API形式は **REST API**（またはServer Actions経由）を基本とする。

### 主要APIエンドポイント（例）

| エンドポイント | メソッド | 用途 |
|:---|:---|:---|
| `/api/contact` | `POST` | 問い合わせデータの送信・保存 |
| `/api/projects` | `GET` | 実績・プロジェクト一覧の取得 |
| `/api/projects/{id}` | `GET` | 特定の実績の詳細取得 |

## 各機能の処理フローと設計

### 1. 問い合わせ処理フロー
```text
ユーザー入力
  ↓
バリデーション (フロント/バックエンド双方)
  ↓
API送信
  ↓
DB保存 (Supabase)
  ↓
通知 (メール / 将来的にはSlack通知)
```

### 2. 実績データ管理
将来のCMS化に伴い、実績データをDBで管理する。
- **管理内容:** プロジェクト名、概要、実績画像（URL）、PDF（URL）等

### 3. PDF・資料管理
- **保存先:** Supabase Storage
- **URL管理:** Storage にアップロードされたファイルのURLをデータベース（Supabase DB）に保存・参照する。

## セキュリティ設計とアクセス制御

### セキュリティ設計
- **入力バリデーション:** フロントエンドおよびサーバー側での厳格な検証
- **スパム対策:** `reCAPTCHA` 等の導入
- **通信設定:** 全通信の `HTTPS` 強制

### アクセス制御
- **管理機能（将来）:** CMS等のデータ更新・削除を行う機能には **Supabase Auth** による認証を必須とする。

---

## ログ管理・パフォーマンス設計

### ログ管理
- **ログ対象:** APIアクセスログ、エラーログ、問い合わせ送信履歴
- **ログ保存先:** Supabase データベース（専用テーブル）や外部ログ基盤

### パフォーマンス設計
- **APIレスポンス最適化:** Next.jsによるキャッシュ戦略（データキャッシュ、Route Cacheの活用）
- **エラーハンドリング:** APIエラーの適切な管理と、フロントエンドを通じたユーザーへの親切な通知処理

---

## 拡張予定機能

バックエンドはこの構想に基づき、今後の段階的な機能追加をサポートする。
- **CMS:** 実績データの登録・編集ダッシュボード（管理画面）の開発
- **AI連携:** 問い合わせ内容のAI分析、AIによるコンテンツ自動生成アシスト等のサービス連携
