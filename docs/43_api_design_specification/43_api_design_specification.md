# SocialBoost 公式Webサイト API設計書

---

## 文書目的

本書は SocialBoost 公式WebサイトにおけるAPI仕様を定義する。

本サイトは基本的にはコーポレートサイトだが、以下の機能を実装するためAPI設計を行う。
- お問い合わせ送信
- 実績データ取得
- 実績詳細取得
- 将来のCMS管理（実績データのCRUD）
- 資料（PDF）管理

API設計を明確にすることで、フロントエンド、バックエンド、データベース間の整合性を確保する。

本書は以下の設計書と連携する。
- バックエンドアーキテクチャ設計書
- データベース設計書
- セキュリティ設計書
- フロントエンドアーキテクチャ設計書

## API設計方針

- **REST API** を採用する
- データフォーマットは **JSON形式** とする
- すべての通信を **HTTPS通信** とする
- 認証が必要なAPI（管理用）と不要なAPI（制約付き公開用）を明確に分離する

## APIエンドポイント仕様（公開API）

### 問い合わせ送信API
| 項目 | 内容 |
|:---|:---|
| **メソッド / エンドポイント** | `POST /api/contact` |
| **用途** | 問い合わせフォーム送信処理 |
| **リクエスト（Body）** | `name`, `email`, `phone`, `company`, `message` |
| **レスポンス** | `status`, `message` |

### 実績一覧取得API
| 項目 | 内容 |
|:---|:---|
| **メソッド / エンドポイント** | `GET /api/projects` |
| **用途** | 実績一覧表示用データの取得 |
| **レスポンス（配列内要素）** | `project_id`, `title`, `summary`, `thumbnail`, `category` |

### 実績詳細取得API
| 項目 | 内容 |
|:---|:---|
| **メソッド / エンドポイント** | `GET /api/projects/{id}` |
| **用途** | 特定の実績詳細ページのデータ取得 |
| **レスポンス** | `project_id`, `title`, `description`, `images`, `pdf_url`, `created_at` |

## レスポンス・ステータス設計

### レスポンス仕様（JSON）
**成功レスポンス例：**
```json
{
  "status": "success",
  "data": { ... }
}
```

**エラーレスポンス例：**
```json
{
  "status": "error",
  "message": "エラー内容の説明"
}
```

### HTTPステータスコード
- `200` : 成功（OK）
- `400` : 不正リクエスト（Bad Request / バリデーションエラー等）
- `401` : 認証エラー（Unauthorized）
- `404` : データなし（Not Found）
- `500` : サーバーエラー（Internal Server Error）

## バリデーション・セキュリティ・制御

### バリデーションルール
- 必須項目チェック
- メール形式（フォーマット）チェック
- 文字数制限（各フィールドの最大/最小文字数）

### セキュリティとスパム対策
- HTTPS必須
- スパム対策（例：`reCAPTCHA`トークンの検証）

### レート制限（Rate Limiting）
- **問い合わせAPI:** `1分間に5回まで`（DDoSや大量のスパム送信を防止）

### ログ管理とパフォーマンス
- **ログ対象:** APIアクセス履歴、エラーログ、問い合わせ送信日時
- **ログ保存先:** Supabase (DBの専用ログテーブル等)
- **パフォーマンス最適化:** CDNを活用したキャッシュの適用（GETリクエスト等）
