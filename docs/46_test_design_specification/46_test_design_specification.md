# SocialBoost 公式Webサイト テスト設計書

---

## 文書目的

本書は SocialBoost 公式Webサイトの品質を保証するためのテスト方針、テスト対象、テスト手法を定義する。

本サイトは単なるコーポレートサイトではなく、
- 高品質UI
- 高速パフォーマンス
- SEO最適化
- モダンアニメーション
- 拡張可能なアーキテクチャ

を持つため、体系的なテスト設計を行う。

本書では以下を定義する。
- テスト対象
- テスト種類
- テスト手法
- 品質基準

本書は以下の設計書と連携する。
- 要件定義書

- 詳細設計書

## テスト方針

1. ユーザー体験を最優先に検証する
2. UIの崩れがないことを全ページ・全デバイスで確認する
3. 高速表示（Core Web Vitals）を基準値で検証する
4. SEO要件を満たすことを確認する
5. ランタイムエラーの発生を徹底的に防ぐ

## テスト種類と実施内容

### 1. 単体テスト（Unit Test）
コンポーネント単位での動作を検証する。
- **対象:** Reactコンポーネント（`Button`, `Card`, `ContactForm` 等）
- **テスト内容:** 正しい表示、クリック動作、propsの受け渡し動作

### 2. 結合テスト（Integration Test）
API連携など複数コンポーネント間の動作を検証する。
- **対象:** API連携処理
- **確認内容:** 問い合わせ送信フロー、実績データ取得と表示の連携

### 3. E2Eテスト（End-to-End Test）
ユーザーの操作導線全体を通した動作を検証する。
- **基本導線:**
  `トップページ → サービス → 実績 → お問い合わせ`

### 4. UIテスト（Visual Test）
デザイン通りに画面が描画されているかを確認する。
- **確認項目:** レイアウト、フォント、カラー、余白

### 5. レスポンシブテスト
各デバイス幅でのレイアウト崩れがないかを検証する。

| デバイス | 代表的なビューポート幅 |
|:---|:---|
| PC | 1920px |
| タブレット | 768px |
| スマートフォン | 375px |

### 6. SEOテスト
検索エンジン最適化の要件が満たされているかを確認する。
- **確認項目:** `meta` タグ、OpenGraph、構造化データ、ページタイトル

## ブラウザ・エラーテスト

### ブラウザテスト（クロスブラウザ対応）
以下の主要ブラウザで表示・動作の確認を行う。
- Chrome
- Safari
- Edge
- Firefox

### エラーテスト
予期されるエラーケースに対して、ユーザーへの適切なフィードバックが表示されることを確認する。
- **入力エラー:** バリデーション失敗時のメッセージ表示
- **APIエラー:** サーバー側エラー時のフォールバック表示
- **ネットワークエラー:** オフラインや通信不良時の挙動確認

### ログ確認
エラー発生時にサーバー側のログに正しく記録されることを確認する。
