# SocialBoost 公式Webサイト 情報設計書（Information Architecture）

---

## 文書目的

本書は SocialBoost 公式Webサイトの情報構造、ページ階層、ナビゲーション設計、URL構造を定義する。

情報設計はUXの基盤であり、
- 迷わないサイト構造
- 理解しやすいページ階層
- SEOに強いURL設計
- 拡張可能なサイト構造

を実現するために重要である。

本書では以下を定義する。
- サイト構造
- ページ階層
- URL設計
- ナビゲーション構造
- コンテンツ配置ルール

本書は以下の設計書と連携する。
- UX設計書
- 画面設計書
- SEO設計書
- コンテンツ設計書
- マーケティング設計書

## サイト構造と階層設計

### 階層構造

| レベル | ページ |
|:---|:---|
| **Level 1** | トップページ |
| **Level 2** | サービス一覧 / 料金 / 実績一覧 / お問い合わせ |
| **Level 3** | サービス詳細 / 実績詳細 |

最大3階層とし、ページ階層を浅く保つ。

### サイトマップ（URL構造）

| URL | ページ名 |
|:---|:---|
| `/` | トップページ |
| `/services` | サービス一覧 |
| `/services/web-design` | Web制作 |
| `/services/lp` | LP制作 |
| `/services/system` | Webシステム開発 |
| `/services/app` | アプリ開発 |
| `/services/ai` | AI導入支援 |
| `/pricing` | 料金ページ |
| `/works` | 実績一覧 |
| `/works/{project}` | 実績詳細 |
| `/contact` | お問い合わせ |

## ナビゲーション設計

### グローバルナビゲーション（ヘッダー）
- サービス
- 料金
- 実績
- お問い合わせ

### フッターナビゲーション
- サービス
- 料金
- 実績
- お問い合わせ

### パンくずリスト
全ページにパンくずリストを実装し、ユーザーと検索エンジンに現在位置を明示する。

**例：**
- `トップ > サービス > Web制作`
- `トップ > 実績 > 実績詳細`

## 本書で確定する事項

| No. | 決定事項 |
|:---|:---|
| 1 | シンプルな最大3階層構造を採用する |
| 2 | サービスページは `/services/{slug}` のカテゴリ構造にする |
| 3 | 実績ページは一覧（`/works`）と詳細（`/works/{project}`）に分ける |
| 4 | パンくずリストを全ページに実装する |
| 5 | SEOを考慮した短く分かりやすいURL設計を行う |
| 6 | 現時点では実装しない |
