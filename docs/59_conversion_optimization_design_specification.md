# SocialBoost 公式Webサイト コンバージョン最適化設計書

---

## 文書目的

本書は SocialBoost 公式Webサイトにおけるコンバージョン最適化（CRO）の設計方針を定義する。

本サイトの目的は単なる情報提供ではなく、
- 問い合わせ獲得
- 営業機会創出
- 信頼形成

である。そのためユーザーがサイト訪問後に自然な流れで相談・問い合わせに至る導線を設計する。

本書ではコンバージョン定義、コンバージョン導線、CTA設計、離脱防止設計、改善プロセスを定義する。

本書は以下の設計書と連携する。
- UX設計書
- Analytics設計書
- マーケティング設計書
- A/Bテスト設計書
- コンテンツ設計書

## コンバージョン定義

### 主要コンバージョン
- **お問い合わせ送信**

### 副次コンバージョン
- お問い合わせページ閲覧
- サービス詳細閲覧
- 実績詳細閲覧
- CTAクリック

---

## コンバージョン導線設計

### 主要導線
`トップページ → サービス → 実績 → お問い合わせ`

### SEO導線
`検索流入 → サービスページ → 料金ページ → お問い合わせ`

### 実績導線
`実績一覧 → 実績詳細 → お問い合わせ`

---

## CTA設計

### ページ別CTA配置

| ページ | CTA配置 |
|:---|:---|
| **トップページ** | ヒーローCTA / ページ下部CTA |
| **サービスページ** | ページ下部CTA |
| **実績ページ** | ページ下部CTA |
| **料金ページ** | 料金説明後CTA |

### CTAコピー例
- `お問い合わせする`
- `相談してみる`
- `無料相談する`

### CTA配置ルール
- **ページ上部:** ヒーローセクション
- **ページ中部:** サービス説明後
- **ページ下部:** 最終CTA

## 問い合わせフォーム最適化

### 入力項目（最小構成）
- 会社名 / 担当者名 / メール / 電話 / 相談内容

### フォーム設計方針
- 入力項目を最小化する
- リアルタイムバリデーションでエラーを即時表示する

---

## 離脱防止と信頼形成設計

### 離脱要因と対策

| 離脱要因 | 対策 |
|:---|:---|
| **情報不足** | サービス詳細の充実 |
| **価格不安** | 料金の考え方を丁寧に説明 |
| **信頼不足** | 制作実績の表示 |

### 信頼形成要素
- 制作実績 / 専門性の提示 / サービス内容の具体性

### 心理的ハードル低減
- 相談歓迎メッセージの掲示
- 無料相談案内
- 小規模案件歓迎の明示

---

## モバイル最適化

- スマートフォンUIの最適化
- CTAボタンの大型化（タップしやすさ確保）

## 改善プロセスとKPI

### 改善プロセス
```text
データ分析 → 課題特定 → 改善案作成 → A/Bテスト → 改善反映
```

### 目標KPI

| 指標 | 目標値 |
|:---|:---|
| **月間問い合わせ数** | 10件 |
| **CVR** | 1% |
| **CTAクリック率** | 追跡・改善 |
